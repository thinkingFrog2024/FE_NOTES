### 浏览器进程模型

浏览器的进程主要有三个

1. 浏览器进程
2. 网络进程
3. 渲染进程
   进程共享一个内存空间
   一个进程崩溃 并不会影响另一个进程

而浏览器进程就可以开启子线程 处理不同任务

渲染进程开启之后 会打开一个主线程 来执行HTML Css JS 而且浏览器会给每个标签页开辟一个新的渲染进程

但是这种方式会消耗大量的内存 以后可能修改策略：一个站点一个进程 。

### 怎么理解js线程和渲染线程的关系？

我们可以理解成：渲染线程和js线程是互斥的：两者不能同时执行。
也可以理解成渲染和js执行是在同一个线程进行的，这是底层的设计，我们只需要明确：js的执行 浏览器的渲染 必然是互斥的。

这是因为：js的执行能够导致重新渲染 而浏览器页面响应事件的时候也能导致js的执行，也就是：两者相互影响。这样的关系决定他们必须是互斥关系。

### 为什么js需要异步机制

假设js不是异步机制，可以想像：如果设置一个五秒之后的定时器，那么在这五秒之内，js线程将会持续等待，直到定时器回调函数执行完毕，在这期间js执行，浏览器无法渲染，用户也无法交互。
（PS: 需要注意以上说法并不绝对 我们知道主线程需要运行js 计算样式 进行布局 绘制 这些过程的确会被js阻塞，但是在合成的时候，是独立于主线程 在合成线程里面运行的 ，这个线程接受主线程绘制的涂层 进行合成 再显示，而transform opacity 的变化 是不会经过布局 和绘制阶段 只会在合成线程中进行 也就是就算主线程卡死了 通过这两个属性实现的动画依旧可以运行）

所以js必须引入一个异步机制：能够让js线程交出控制权，并且还能在合适的时机重新获取控制权。

在node.js里面 存在一个线程，我们叫这个线程watchdog,js也是类似的： 存在一个定时器的触发线程 。

还是以定时器举例：在设置了定时器之后，js发现现在没有其他的任务需要运行，只有一个到点了需要执行的回调函数。

于是js通知watchdog：我设置了一个定时器 到点了需要执行回调函数 你可以把这个回调函数包装成一个任务 放到消息队列的末尾 等主线程空闲的时候 通知我。

于是这五秒的时间 浏览器就可以正常响应用户，在浏览器的每帧，js线程也会看看事件对列有没有东西要执行，五秒之后，发现对列有回调函数了，于是js获取线程控制权 开始执行回调函数。

所以何谓异步机制：也就是不一直占用线程控制权

### 浏览器与系统调度cpu

系统可以调度cpu的资源 并且强制某个任务交还cpu的资源。
但是浏览器并不能

在浏览器的eventLoop里面，逻辑就是：执行一个宏任务 执行所有的微任务 要是有空 就进行渲染
但是 浏览器是不能强制js线程交还控制权的 。

虽然我们一般认为浏览器有六十帧，在每帧里面都会进行一遍eventLoop,但是这不是强制的，也就是，如果你的js执行时间太长了 这帧就直接被撑爆了，浏览器将会错过好几次渲染机会 用户会感觉好像卡了一下，这就是所谓的掉帧。

我们可以参考react的分片思想 使用requestIdleCallback在浏览器的空闲时间 执行一些小的任务 在做动画的时候 使用requestAnimationFrame 来进行动画的渲染。
或者我们可以使用web worker  Worker 和主线程之间是 隔离 的，它们通过 消息传递（postMessage） 来沟通。

- 不能操作 DOM ：Worker 线程 绝对不能 碰 document 、 window 、 parent 。它看不见页面，只能做纯计算。想更新 UI？把计算结果发回主线程，让主线程去更。
- 同源策略 ：Worker 脚本必须和主页面同源。
- 数据拷贝 ：主线程和 Worker 之间传数据是 拷贝（Copy） ，不是共享引用。传个 100MB 的对象过去，浏览器得先复制一份，这本身也有开销。如果你真要传大数据，去学学 Transferable Objects （如 ArrayBuffer ），那是零拷贝转移。

#### Transferable Objects

普通的对象传递就像 复印文件 ：主线程有一份 100MB 的数据，要传给 Worker，浏览器得先把这 100MB 完整拷贝一份 ，然后再发过去。这中间的 CPU 和内存开销是巨大的，甚至可能卡死主线程。

```javascript
// 主线程
const data = new ArrayBuffer(100 * 1024 * 1024); // 100MB 数据
const worker = new Worker('worker.js');
worker.postMessage(data, [data]); // 传递数据，同时转移所有权
```

不是所有东西都能转移，只有特定的二进制数据类型才行：

- ArrayBuffer
- MessagePort
- ImageBitmap
- OffscreenCanvas
  只有上面这些底层的二进制接口才支持所有权转移。
  这是因为js对象在内存里面是非常复杂的，如果一个线程需要把这样的数据传递给另外一个线程，由于堆 或者说内存空间的隔离 传递指针地址是不可行的。所以只能深克隆对象 。

但是二进制数据 对应的是一段连续的 裸露的系统内存 ，也就是说 二进制数据并不是存储在v8堆内存 并不由垃圾回收机制管理，不存在复杂的数据结构。

这样的数据 传递的时候 就是传递对应的内存地址，在原进程中 这个数据的地址会被重置
