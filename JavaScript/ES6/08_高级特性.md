## proxy

proxy可以用来修改某些操作的默认行为，等同于在语言层面进行修改，所以属于一种元编程 （也就是对变成语言进行编程）。

可以使用这个代理对象的get set

1. vue里面的数据追踪
2. react里面的不可变状态管理
3. 链式调用
4. 惰性创建：设计模式里面的惰性单例模式可以在实例被调用的时候在创建实例，vue里面的reactive就是在被访问的时候才会创建其代理对象。
5. 数据保护：实现深层的只读对象
6. 有序对象：指定对象的某个属性在遍历的时候有一个特定的顺序 比如设计某个属性一定最后被遍历。

9. getPrototypeOf(target)

12. **apply(target, object, args)**：拦截 Proxy 实例作为函数调用的操作

**get**

get拦截的三个参数依次为目标对象,属性名,proxy实例本身(严格地说其实是操作行为针对的对象 可选).

这个方法是可以继承的,就比如让一个具有get方法的对昂作为另一个对象的原型对象 那么这个对象可以调用这代理对象的get函数





**那么使用Refect对象修改 访问对象的属性和直接访问读取有什么区别?**

* 当使用Reflect对象进行操作时,如果操作失败会返回false 不会直接抛出错误(当使用Reflect对象写入一个不可写的对象时会有这种情况 )
* 当我们在get set陷阱里面继续使用直接赋值,取值,会触发proxy陷阱



**set**

set的第四个参数receiver,指的是 **原始的操作行为所在的那个对象**, **一般情况下就是代理对象本身**,不过当一个设置了set方法的代理对象作为另一个对象的原型时,在触发set的时候,这个receiver就是那个对象 比如:

```javascript
const handler = {
  set: function(obj, prop, value, receiver) {
    obj[prop] = receiver;
    return true;
  }
};
const proxy = new Proxy({}, handler);
proxy.foo = 'bar';
proxy.foo === proxy // true
```

不过对于不可写的属性 set方法也不能修改.

set代理应该 **返回一个布尔值**,在严格模式下 如果返回的不是布尔值 应该会报错.



**apply**

这个方法将会拦截函数的调用,call,apply作用.

```javascript
var handler = {
  apply (target, ctx, args) {
    return Reflect.apply(...arguments);
  }
};
```

第一个参数代表目标对象,目标对象上下文,目标对象的参数数组.

在上面讲到 使用Reflect对象进行get set操作时,会绕过陷阱检查,但是使用apply

不会,如果给他设置了appy陷阱,那么不管是直接调用,call调用,apply调用,Reflect调用,都会经过陷阱检查

**has**

这个方法可以拦截判断对象是否有某个数星星的判断比如in操作符.

我们可以使用这个陷阱隐藏某些属性,比如判断某个下划线开头的属性,使用这个拦截并且返回undefined.

但是如果一个对象是不可配置,不可拓展的 不能使用has拦截 这里会报错.

has拦截的返回结果还是最终取决于我们的代码逻辑,我们可以通过in操作符检查:

但是为什么可以这样呢?要知道in操作符的本质就是HasProperty 而has就是拦截的HasProperty,如果在has陷阱里面调用in应该会形成套娃. 

这其实是因为在has拦截里面使用in操作符,因为in操作符在内部实现的时候会直接调用 `Reflect.has`.

**construct**

这个方法用于拦截new命令 接受三个参数:target args newTarget 

这个方法必须返回一个对象 否则会报错

另外 由于construct拦截的是构造函数 所以目标对象必须是函数 不然就会报错.

一般情况下 我们调用new操作符in时 调用构造函数 这个构造函数里面的this将会指向对象,但是这里调用handler对象内的construct拦截器时将会指向handler对象.

**deleteProperty**

这个方法用于拦截delete命令,如果这个方法抛出错误或者返回false 那么删除操作将会失败.

**this问题**

虽然Proxy可以代理目标对象的访问 但是这个代理对象在不处理的情况下 并不能保证和目标对象的行为一模一样.主要就是因为在proxy'代理的时候 目标对象内部的this将会指向代理对象 而不再是原对象

```javascript
const target = {
  m: function () {
    console.log(this === proxy);
  }
};
const handler = {};

const proxy = new Proxy(target, handler);

target.m() // false
proxy.m()  // true
```

```javascript

  m: function () {
    console.log(this === proxy);
  }
};
const handler = {};

const proxy = new Proxy(target, handler);

target.m() // false
proxy.m()  // true
```

在上面的例子里面 由于this的指向改变了 代理对象 原对象方法的结果就会有所不同.

```javascript
const _name = new WeakMap();

class Person {
  constructor(name) {
    _name.set(this, name);
  }
  get name() {
    return _name.get(this);
  }
}

const jane = new Person('Jane');
jane.name // 'Jane'

const proxy = new Proxy(jane, {});
proxy.name // undefined
```

在这个例子里面由于属性值的保存直接和this'相关 所以使用proxy调用的时候 会出现undefined

在此之外 有一些原生对象的`内部属性 只有正确的this才能拿到 使用proxydialing这些原生对象也会导致错误.可以使用重新绑定this也可以解决问题.

```javascript
const target = new Date('2015-01-01');
const handler = {
  get(target, prop) {
    if (prop === 'getDate') {
      return target.getDate.bind(target);
    }
    return Reflect.get(target, prop);
  }
};
const proxy = new Proxy(target, handler);

proxy.getDate() // 1
```

**Mixin模式**

混入指的是多个对象合成一个新的对象 新的对象具有各个组成成员的接口 最简单的就是使用解构赋值

```javascript
const a = {
  a: 'a'
};
const b = {
  b: 'b'
};
const c = {...a, ...b}; // {a: 'a', b: 'b'}
```

函数实现：把多个类合并成一个类

```javascript
function mix(...mixins){
    class Mix{
        constructor(){
            for(let mixin of mixins){
                copyProperty(this,new mixin())
            }
        }
    }
    //拷贝静态属性 原型属性 
   for (let mixin of mixins) {
    copyProperties(Mix, mixin); // 拷贝静态属性
    copyProperties(Mix.prototype, mixin.prototype); // 拷贝原型属性
  }
    return Mix
}
function copyProperties(target, source) {
  for (let key of Reflect.ownKeys(source)) {
    if ( key !== 'constructor'
      && key !== 'prototype'
      && key !== 'name'
    ) {
      let desc = Object.getOwnPropertyDescriptor(source, key);
      Object.defineProperty(target, key, desc);
    }
  }
}
```

**装饰器**

装饰器就是一种函数 写作@+函数名。

```javascript
//装饰类
@frozen class Foo {
    //装饰方法
  @configurable(false)
  @enumerable(true)
  method() {}

  @throttle(500)
  expensiveMethod() {}
}
```

ts写法：

```javascript
type Decorator = (value: Input, context: {
  kind: string;
  name: string | symbol;
  access: {
    get?(): unknown;
    set?(value: unknown): void;
  };
  private?: boolean;
  static?: boolean;
  addInitializer?(initializer: () => void): void;
}) => Output | void;
```

这个函数的返回值是一个新版本的装饰对象 或者不返回

- `kind`：字符串，表示装饰类型，可能的取值有`class`、`method`、`getter`、`setter`、`field`、`accessor`。
- `name`：被装饰的值的名称: The name of the value, or in the case of private elements the description of it (e.g. the readable name).
- `access`：对象，包含访问这个值的方法，即存值器和取值器。
- `static`: 布尔值，该值是否为静态元素。
- `private`：布尔值，该值是否为私有元素。
- `addInitializer`：函数，允许用户增加初始化逻辑。



装饰器的执行步骤如下。

1. 计算各个装饰器的值，按照从左到右，从上到下的顺序。
2. 调用方法装饰器。
3. 调用类装饰器。

**为什么装饰器不能用于函数？**

上面说到了 装饰器可以用于类 类的方法 但是不能用于函数 因为函数存在提升 函数在定义的时候·装饰器还没有定义

就好比：

```javascript
var counter = 0;

var add = function () {
  counter++;
};

@add
function foo() {
}
```

在变量提升和函数提升的作用下这段代码会变成：

```javascript
var counter;
var add;

@add
function foo() {
}

counter = 0;

add = function () {
  counter++;
};
```

一定要想包装函数的话 在函数外面再包一层函数吧。

**使用装饰器实现自动发布事件**

使用postal lodash版本

```javascript
// 引入postal库，它是一个用于发布/订阅模式的库，这里特别引入了lodash版本的postal
const postal = require("postal/lib/postal.lodash");

// 定义一个函数publish，它将被用作方法装饰器
export default function publish(topic, channel) {
  // 如果没有提供channel参数，则默认channel为'/'
  const channelName = channel || '/';
  // 使用提供的或默认的channelName创建一个消息通道
  const msgChannel = postal.channel(channelName);

  // 在指定的topic上订阅消息，当有消息发布到这个topic时，执行回调函数
  msgChannel.subscribe(topic, v => {
    console.log('频道: ', channelName); // 打印订阅的频道名称
    console.log('事件: ', topic);        // 打印订阅的事件topic
    console.log('数据: ', v);            // 打印接收到的数据
  });

  // 返回一个函数，这个函数将在类的属性定义时被调用
  return function(target, name, descriptor) {
    // 保存原始方法的引用
    const fn = descriptor.value;

    // 修改descriptor.value，实际上是修改了方法的实现
    descriptor.value = function() {
      // 调用原始方法，并保存其返回值
      let value = fn.apply(this, arguments);
      // 使用msgChannel.publish将返回值发布到指定的topic
      msgChannel.publish(topic, value);
    };
  };
}

class MyService {
  @publish('myService:event', 'myChannel')
  myMethod() {
    return { data: 'This is a test' };
  }
}

const service = new MyService();
service.myMethod(); // 这将调用myMethod，并将其返回值发布到'myChannel'的'myService:event' topic

// index.js
import publish from './publish';

class FooComponent {
  @publish('foo.some.message', 'component')
  someMethod() {
    return { my: 'data' };
  }
  @publish('foo.some.other')
  anotherMethod() {
    // ...
  }
}

let foo = new FooComponent();

foo.someMethod();
foo.anotherMethod();
```

## module

 ES6模块自动开启严格模式，主要有以下限制：

- 变量必须声明后再使用
- 函数的参数不能有同名属性，否则报错
- 不能使用`with`语句
- 不能对只读属性赋值，否则报错
- 不能使用前缀 0 表示八进制数，否则报错
- 不能删除不可删除的属性，否则报错
- 不能删除变量`delete prop`，会报错，只能删除属性`delete global[prop]`
- `eval`不会在它的外层作用域引入变量
- `eval`和`arguments`不能被重新赋值
- `arguments`不会自动反映函数参数的变化
- 不能使用`arguments.callee`
- 不能使用`arguments.caller`
- 禁止`this`指向全局对象
- 不能使用`fn.caller`和`fn.arguments`获取函数调用的堆栈
- 增加了保留字（比如`protected`、`static`和`interface`）

**export命令**

规定模块的对外接口 对外接口必须和模块内部的变量建立对应关系。

```javascript
// 报错
export 1;

// 报错
var m = 1;
export m;

// 写法一
export var m = 1;

// 写法二
var m = 1;
export {m};

// 写法三
var n = 1;
export {n as m};
```

export 可以对外输出 函数 类 var let const 声明的变量。

并且export输出的接口和值是动态绑定关系，可以体哦那个锅接口 取到模块内部实时的值。commonJS输出的是值的缓存 不存在动态更新。

**import**

`import`命令输入的变量都是只读的，因为它的本质是输入接口。也就是说，不允许在加载模块的脚本里面，改写接口。比如a是一个基本类型，修改就会报错，但是如果a是一个对象 那么修改对象的属性是可以的。 **但是在其他模块里面的对象属性是也会改变的** 所以不建议修改接口。

import后面可以是相对路径 绝对路径 模块名 

import具有提升作用 会提升到整个模块的头部

```javascript
foo();

import { foo } from 'my_module';
```

由于import是静态执行 不能使用表达式变量 （这些在运行的时候才能得到结果 import需要在编译时就知道结果）

```javascript
// 报错
import { 'f' + 'oo' } from 'my_module';

// 报错
let module = 'my_module';
import { foo } from module;

// 报错
if (x === 1) {
  import { foo } from 'module1';
} else {
  import { foo } from 'module2';
}
```

使用import执行某个模块：

```javascript
import 'lodash';
```

会执行这个模块里面的代码 包括初始化代码 在模块顶层声明的代码，如果lodash这个模块的执行会产生副作用，比如修改全局变量，或者注册全局函数 ，如果多次重复执行同一句import，只会执行一次，就比如：

```javascript
import { foo } from 'my_module';
import { bar } from 'my_module';

// 等同于
import { foo, bar } from 'my_module';
```



通过Babel转码 CommonJS模块的require命令 ES6的import命令 可以写在同一个模块里面 但是这样可能引起问题：import引入的模块如果依赖于require的模块 就会有顺序错误。



**模块的整体加载**

前面提到了指定加载模块的某个输出值 我们也可以把这个模块整个加载过来 就是用*指定一个对象 所有输出值都会加载在这个对象上。

```javascript
import * as circle from './circle';
```

这个指定的对象要在编译（其实是一个预处理阶段）的时候静态分析（这意味着导入的模块在编译时就确定了解构和可以使用的成员 比如在某个条件判断里面修改了这个对象 那么这个对象就可能有不同的结果）

预处理都会做些什么？

1. 解析模块依赖：解析export import ，确定模块之间的依赖关系。

2. 静态分析：分析检查模块的导出成员

3. 代码转换：如果需要的话 可以把es6模块转换成CommonJS模块

4. 模块打包：打包工具进行

5. 作用域：确定每个模块的作用域是隔离的 防止全局变量污染

6. 代码优化：tree-shaking 压缩代码等等

7. 生成模块映射：生成模块之间的映射关系 在运行的时候可以正确的加载解析模块

   ES6的模块隔离是通过加载机制实现的 每个模块都会在自己的上下文里面执行 所以引入模块就相当于添加了全局变量

   生成模块映射指的是创建一个数据解构，用于记录模块之间的依赖关系以及模块资源的路径标识符



**export default**

这个东西就是为了不用阅读文档 可以快速加载模块 

```
export default function () {
  console.log('foo');
}
```

其他模块加载这个模块的时候 import可以为这个默认=导出的变量/函数指定任意名字。这个时候导出导入都不需要大括号，

一个模块只能由一个默认输出 这样才能确保  本质上就是导出了一个叫做default的变量或者方法 其实就相当于：

```javascript
// modules.js
function add(x, y) {
  return x * y;
}
export {add as default};
// 等同于
// export default add;

// app.js
import { default as foo } from 'modules';
// 等同于
// import foo from 'modules';
```

export default a的含义是把变量a的值赋给default 所以后面应该是一个变量 而不是一个赋值语句 之前说到export后面不可以世界是一个值 而是一个接口 而export default 可以 



**export import的复合写法**

如果在一个模块里面先输入再输出一个模块 那么 import export可以写在一起。

export { foo, bar } from 'my_module';

import { foo, bar } from 'my_module';

//默认接口

这样写 foo bar实际上并没有被导入当前模块

**import.meta**

ES2020给import命令添加了一个元属性 `import.meta` 返回当前模块的元信息 这个属性返回一个对象 这个对象的各种属性就是当前脚本的各种信息 具体有哪些属性标准没有规定 一般有下面两个属性

**module的加载实现**

传统的加载方式 通过script加载本地或者外部脚本，如果加载的是外部脚本，而且体积很大，就需要异步加载 有两种异步加载方式：

1. defer `<script src="path/to/myModule.js" defer></script>` 
2. async `<script src="path/to/myModule.js" async></script>`

打开这两个属性 脚本就会异步加载 区别是:defer需要等到整个页面在内存里面正常渲染结束 （dom解构生成 其他脚本执行完成） 而 async 一旦下载完了就会立即执行 中断dom渲染defer是可以保证下载顺序的 async就不能了。

**加载规则**

es6模块的type是module（es6模块的典型特征就是使用了import import（） export）

浏览器对于es6模块 都是异步加载 不会造成堵塞 相当于打开了defer 但是也可以手动打开async

也支持内嵌在网页里面

对于外部的模块脚本 有几点需要注意

1. 引入的脚本相当于在模块作用域之内运行的 这个很明显吧 作用域的问题
2. 自动开启严格模式
3. 模块内 import的时候 .js后缀 **不可以省略** 



**ES6模块与CommonJS模块的差异**

这两个模块有三个重大差异

1. CommonJS模块输出的是一个值的拷贝 ES6模块输出的是值的引用。
2. CommonJS模块是运行时 
3. CommonJS是同步加载 ES6是异步的 有一个独立的模块依赖的解析过程。

第二个差异是因为 CommonJS 加载的是一个对象（即`module.exports`属性），该对象只有在脚本运行完才会生成。而 ES6 模块不是对象，它的对外接口只是一种静态定义，在代码静态解析阶段就会生成。
