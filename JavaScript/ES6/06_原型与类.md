# 第六章：原型与类

---

## 1. 原型与继承 (Prototypal Inheritance)

### 1.1 `[[Prototype]]`：隐藏的血脉
每个对象都有一个隐藏属性 `[[Prototype]]`。这就像是你的基因，决定了你从哪里来。
*   它要么是 `null`，要么是对另一个对象的引用（原型对象）。
*   **继承规则**：当你访问一个属性，而你自己没有时，JS 引擎会顺着这根血脉往上找。这就是“原型继承”。

### 1.2 操作血脉的方式
亲爱的，虽然血脉是隐藏的，但我们总有办法去操纵它。

1.  **`__proto__` (The Old Way)**
    *   这是历史遗留的产物。ES5 引入了 `Object.getPrototypeOf` / `Object.setPrototypeOf` 来代替它，但它依然顽强地活在浏览器里。
    *   `rabbit.__proto__ = animal`：给 `rabbit` 接上了 `animal` 的血脉。
    *   **限制**：
        1.  只能是对象或 `null`。
        2.  不能形成闭环（不能既是父亲的儿子，又是父亲的父亲）。
    *   > **Darling's Note**: `__proto__` 只是 `[[Prototype]]` 的 getter/setter。当你修改它时，其实是在修改那个隐藏的内部属性。

2.  **`Object.create(proto, [descriptors])`**
    *   这是更优雅的方式。凭空创造一个对象，并指定它的祖先。
    *   `let rabbit = Object.create(animal)`：这等同于 `{__proto__: animal}`。
    *   **深度克隆**：`Object.create(Object.getPrototypeOf(obj), Object.getOwnPropertyDescriptors(obj))`。这才是真正的克隆，连血统和性格（属性描述符）都复制得一模一样。

## 2. 原型简史 (A Brief History)

你想知道这一切是怎么发生的吗？来听听这段历史。

*   **远古时代**：构造函数的 `prototype` 属性一直都在。这是最古老的方式。
*   **2012年**：`Object.create` 出现了。它能创建对象，但不能修改已有的原型。于是大家私下里都用非标准的 `__proto__`。
*   **2015年 (ES6)**：`Object.setPrototypeOf` 和 `Object.getPrototypeOf` 终于转正了。但 `__proto__` 因为用的人太多，也被迫写进了标准（虽然只在浏览器环境必选）。
*   **2022年**：官方终于允许在对象字面量 `{ __proto__: ... }` 中使用它，但依然不推荐作为 getter/setter 使用。

### 为什么 `__proto__` 是个坏东西？
看这个例子，我的小傻瓜：
```javascript
let obj = {};
let key = prompt("What's the key?", "__proto__");
obj[key] = "some value"; 
alert(obj[key]); // [object Object]，并不是 "some value"！
```
如果你允许用户输入键名，而用户输入了 `__proto__`，并且赋值了一个非对象（比如字符串），这个赋值会被**忽略**！
因为原型只能是对象或 null。这会导致严重的安全漏洞（原型污染）。

**解决方案**：
1.  使用 `Map` 代替对象。
2.  创建一个没有原型的对象：`Object.create(null)`。这种对象叫 **Very Plain Object**，它没有 `__proto__` 属性，所以怎么改都安全。

## 3. 原型链的细节 (Details)

### 3.1 写入与屏蔽
*   **读取**：顺着原型链往上找。
*   **写入/删除**：直接在对象本身操作。
    *   `rabbit.walk = function() {}`：这会在 `rabbit` 上创建自己的方法，而不会修改 `animal` 的方法。这叫**属性屏蔽 (Shadowing)**。

### 3.2 访问器属性 (Accessors) 的例外
但是！如果原型上有一个 setter，写入操作**不会**覆盖它，而是会调用这个 setter。

```javascript
let user = {
  name: "John",
  surname: "Smith",
  set fullName(value) {
    [this.name, this.surname] = value.split(" ");
  },
  get fullName() {
    return `${this.name} ${this.surname}`;
  }
};

let admin = {
  __proto__: user,
  isAdmin: true
};

admin.fullName = "Alice Cooper"; // 触发 setter！
```
**关键点**：`this` 依然指向 `admin`（点符号前的对象）。
所以，虽然调用的是原型上的 setter，但属性（`name`, `surname`）是写在 `admin` 身上的。
*   `admin` 有了自己的 `name` 和 `surname`。
*   `user` 的属性毫发无损。
这就是**方法共享，状态独立**。

### 3.3 `for..in` 循环
*   `for..in` 会遍历**所有**可枚举属性，包括**继承**来的。
*   如果你只想看自己的属性，用 `obj.hasOwnProperty(key)` 过滤一下。
    *   *Prompt*: `hasOwnProperty` 是哪里来的？它是 `Object.prototype` 上的方法。为什么它没被遍历出来？因为它默认是不可枚举的。

## 4. 构造函数与 `F.prototype`

```javascript
function Rabbit(name) {
  this.name = name;
}
Rabbit.prototype = animal; 
let rabbit = new Rabbit("White Rabbit"); // rabbit.__proto__ == animal
```

### 4.1 仅在 `new` 时生效
`F.prototype` 只有在 `new F()` 的那一瞬间，会被赋值给新对象的 `[[Prototype]]`。
之后你再改 `F.prototype`，已经出生的对象是不会变的。就像你不能改变已经发生的事实。

### 4.2 默认的 `constructor`
每个函数天生都有 `prototype`，里面天生有一个 `constructor` 指向自己。
`Rabbit.prototype.constructor === Rabbit`。

**用途**：
我们可以用它来创建一个“同类”对象，即使不知道构造函数是谁。
```javascript
let rabbit2 = new rabbit.constructor("Black Rabbit");
```
**风险**：
如果你粗暴地重写了 `prototype`：`Rabbit.prototype = {}`，这个 `constructor` 就没了。
所以，请温柔一点：`Rabbit.prototype.jumps = true`（添加属性，而不是覆盖对象）。

## 5. 类 (Class)：优雅的契约

### 5.1 基础语法
```javascript
class MyClass {
  // 类字段 (Class Fields)
  name = "My Class";

  constructor() { ... }
  
  method() { ... } // 原型方法
  
  get prop() { ... } // Getter
  set prop(v) { ... } // Setter
}
```
**本质**：
*   `class` 是函数的变体。`typeof MyClass === 'function'`。
*   `MyClass === MyClass.prototype.constructor`。
*   方法挂在 `MyClass.prototype` 上。

**区别**：
1.  必须用 `new` 调用。
2.  类方法不可枚举。
3.  默认严格模式。
4.  内部属性 `[[IsClassConstructor]]: true`。

### 5.2 类字段 (Class Fields)
`name = "John"`。

亲爱的，别被“类字段”这个名字骗了。它**不在**类上，也**不在**原型上。
它在**每个实例**上。

> **Daddy's Analogy**:
> *   **原型方法 (method)**：就像家族的**图书馆**。所有孩子共享同一个图书馆，同一本书。
> *   **类字段 (field)**：就像给每个孩子发的**私房钱**。每个孩子手里都有一份，互不干扰。
> 
> ```javascript
> class User {
>   name = "John"; // 相当于在 constructor 里写 this.name = "John"
> }
> let u1 = new User();
> let u2 = new User();
> u1.name = "Alice"; 
> // u2.name 依然是 "John"，因为它们是独立的！
> ```

*   它在 `super()` 返回后，构造函数逻辑执行前初始化。
*   **用途**：绑定方法的 `this`。
    ```javascript
    class Button {
      click = () => {
        alert(this.value); // 箭头函数捕获了 Button 实例的 this
      }
    }
    ```

## 6. 继承与 `super`

### 6.1 `extends`
`class Rabbit extends Animal`。
*   `Rabbit.prototype.[[Prototype]]` -> `Animal.prototype`（实例方法的继承）。
*   `Rabbit.[[Prototype]]` -> `Animal`（静态方法的继承）。

### 6.2 `super` 的陷阱
1.  **构造函数**：继承类的 `constructor` 必须先调用 `super()`。
    *   **注意**：如果派生类没有定义 `constructor`，JS 引擎会自动生成一个默认的 `constructor(...args) { super(...args); }`。你不需要显式编写它。
    *   **但是**，如果你显式写了 `constructor`，就**必须**在其中调用 `super()`。
    *   因为继承类（Derived）不创建 `this`，它指望父类创建。
    *   如果不调，`this` 就是未定义的。
2.  **字段初始化顺序**：
    *   基类：字段初始化 -> 构造函数。
    *   子类：`super()` -> 父类构造 -> 字段初始化 -> 子类构造。
    *   **后果**：如果在父类构造函数里调用被子类重写的方法/字段，可能会访问到未初始化的值。

### 6.3 `[[HomeObject]]`：super 的灵魂
这是理解 `super` 的关键。
当一个函数被定义为类或对象的方法时，它会获得一个内部属性 `[[HomeObject]]`，指向它所属的对象。
`super` 使用 `[[HomeObject]]` 的原型来查找父类方法。
*   它不依赖 `this`。
*   它是固定的，不能改变。
*   **注意**：只有标准的方法语法 `method() {}` 会有这个属性，`method: function() {}` 没有！

## 7. 静态 (Static) 与 私有 (Private)

### 7.1 静态成员
`static method()`。
*   属于类，不属于实例。
*   **继承**：子类可以继承父类的静态方法（通过原型链）。

### 7.2 静态块 (Static Block) (ES2022)
```javascript
class C {
  static x = ...;
  static {
    // 这里可以写复杂的初始化逻辑
    // 可以访问私有属性！
    this.x = doSomething();
  }
}
```
它在类定义时执行一次。

### 7.3 私有成员 (`#`)
*   **绝对私密**：真正的私有，JS 引擎层面的强制隔离，外面的人绝对进不去。
*   **禁止偷窥**：不能通过 `this['#name']` 访问。。
*   **血统纯正**：子类无法访问父类的私有成员。
*   **In Operator**：`#field in obj` 可以检查私有字段是否存在（ES2022）。就像在问：“这里面有点东西吗？”

#### 为什么要有私有成员？
1.  **保护敏感部位**：防止外部脏手乱摸内部状态。
2.  **隐藏实现细节**：外面只需要知道怎么用（Public API），不需要知道里面怎么动（Private Implementation）。
3.  **防止意外**：如果所有属性都是公开的，随便谁都能改，对象的状态很容易被玩坏。

## 8. 内建类的扩展

### 8.1 继承原生类
`class PowerArray extends Array`。
它会继承 `Array` 的所有方法（`map`, `filter` 等）。

### 8.2 `Symbol.species`
**控制繁衍后代的品种**。

简单来说：**当你用 `map` / `filter` 这种方法生成新数组时，你想得到什么？**
1.  **默认情况**：你是什么类，生出来的孩子就是什么类。
    *   你是 `PowerArray`，生出来的也是 `PowerArray`。
    *   这通常很棒，因为孩子也能用父类的超能力（方法）。
2.  **`Symbol.species` 的作用**：有时候你不想让孩子继承这种特殊的身份。
    *   你想让 `map` 出来的结果只是一个普普通通的 `Array`。
    *   这时候你就用 `Symbol.species` 告诉 JS：“别给我生 `PowerArray` 了，给我生个普通 `Array` 就行。”

```javascript
class PowerArray extends Array {
  isEmpty() { return this.length === 0; }

  // 强制绝育（或者说基因清洗）：
  // 告诉 JS 引擎：当这个类的实例进行 map/filter 等操作时，
  // 生出来的新对象应该只是普通的 Array，而不是 PowerArray。
  static get [Symbol.species]() {
    return Array;
  }
}

let a = new PowerArray(1, 2, 3);
let b = a.map(x => x * x); // b 是普通的 Array，不是 PowerArray

// 如果没有上面的 species 设置，b 也会是 PowerArray，也能调用 isEmpty
// 但现在，它只是个普通数组，所以...
console.log(b.isEmpty()); // Error! b 只是普通数组，没有这种特殊服务
```

### 8.3 静态方法不继承 (内建类的特殊癖好)
通常情况下，子类会继承父类的静态方法。
*   `class Son extends Daddy`，那么 `Son.makeMoney()` 也是可以的，就像子承父业。

**但是！内建类（Built-in Classes）是个例外。**
JS 的内建对象之间虽然有继承关系（`Array` 确实继承自 `Object`），但它们的**静态方法不继承**。
*   `Object.keys()` 存在。
*   `Array` 继承自 `Object`。
*   但 `Array.keys()` **并不是**继承自 `Object.keys()` 的。
    *   实际上 `Array.keys()` 返回的是迭代器，和 `Object.keys()` 功能完全不同。
*   再比如 `Object.assign()` 存在，但 `Array.assign()` **根本不存在**。

**为什么？**
为了保持血统纯正和避免混乱。如果 `Array` 继承了 `Object` 所有的静态工具方法（比如 `assign`, `getPrototypeOf` 等），那 `Array` 这个类本身就会变得非常臃肿且奇怪。JS 引擎选择让它们在静态层面“断绝父子关系”。

## 9. 类型检查 (Type Checking)

### 9.1 `instanceof`
`obj instanceof Class`。
*   逻辑：检查 `Class.prototype` 是否在 `obj` 的原型链上。
*   **自定义**：如果有静态方法 `[Symbol.hasInstance]`，直接调用它。
    ```javascript
    class Animal {
      static [Symbol.hasInstance](obj) {
        return !!obj.canEat; // 只要能吃，就是 Animal
      }
    }
    ```

### 9.2 `Object.prototype.toString`
`Object.prototype.toString.call(obj)`。
*   返回 `[object Type]`。
*   **自定义**：对象可以有 `[Symbol.toStringTag]` 属性。
    ```javascript
    let user = { [Symbol.toStringTag]: "User" };
    alert({}.toString.call(user)); // [object User]
    ```

## 10. 混入 (Mixins) 与 `new.target`

### 10.1 `new.target`：谁在碰我？
当你用 `new` 调用一个构造函数时，`new.target` 属性会指向那个被调用的构造函数。
*   这就像是你在问：“是谁在让我动？”
*   如果不是通过 `new` 调用的，它是 `undefined`。
*   **重要用途**：创建**抽象类**（Abstract Class）。
    *   你可以规定：“我是个抽象的父类，你不能直接碰我（实例化我），你只能碰我的孩子（子类）。”

```javascript
class Animal {
  constructor() {
    if (new.target === Animal) {
      throw new Error("Ah... you can't touch me directly! Use my children instead.");
    }
    console.log(`I was created by ${new.target.name}`);
  }
}

class Rabbit extends Animal {}

new Rabbit(); // Works! "I was created by Rabbit"
new Animal(); // Error! "Ah... you can't touch me directly!"
```

### 10.2 混入模式 (Mixins)
JS 只支持单继承（只能有一个爸爸）。但有时候，你想要拥有多个人的特长。
这时候就需要 **混入 (Mixins)**。
*   Mixin 不是类，它只是一个包含方法的对象。
*   你可以把这些方法“混入”（复制）到你的原型里。
*   这就像是... *多P*。虽然你只能嫁给一个人（继承），但你可以和很多人发生关系（混入），学会他们的技巧。

```javascript
// 定义一些特殊的技巧
let sayHiMixin = {
  sayHi() {
    console.log(`Hello ${this.name}`);
  }
};

let sayByeMixin = {
  sayBye() {
    console.log(`Bye ${this.name}`);
  }
};

class User {
  constructor(name) {
    this.name = name;
  }
}

// 把技巧复制到 User 的身体里
Object.assign(User.prototype, sayHiMixin, sayByeMixin);

new User("Dude").sayHi(); // Hello Dude!
```
