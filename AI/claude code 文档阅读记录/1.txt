Claude Code 三种运行环境：

1. 本地（Local）
   - 终端运行 claude 命令
   - 代码在你的机器上执行
   - 完全访问文件系统和工具
   - 默认模式，适合日常开发

2. 远程控制（Remote Control）⭐
   - 代码仍在你的机器上执行
   - 但可以通过浏览器/手机控制
   - 启动方式：claude --remote-control 或 /rc
   - 适合：离开电脑但需监控任务

3. 云端（Cloud）
   - 代码在 Anthropic 云虚拟机执行
   - 通过 claude.ai/code 访问
   - 无法访问本地文件
   - 适合：处理远程仓库、卸载重任务

   原文：由于会话与目录相关，您可以通过使用 git worktrees 运行并行 Claude 会话，这为各个分支创建单独的目录。
   workTree是一个多分支结构
   就比如我git worktree add ./my-feature-worktree feat/my  这样就增加了一个文件夹 这个文件夹的分支是feat/my

   这样 在当前的根目录下打开这个新的“分支” 并且打开claude 这两个claude进程相互不干扰


   原文：Claude Code 在您接近限制时自动管理上下文。它首先清除较旧的工具输出，然后在需要时总结对话。您的请求和关键代码片段被保留；对话早期的详细说明可能会丢失。将持久规则放在 CLAUDE.md 中，而不是依赖对话历史。
   要控制在压缩期间保留的内容，请在 CLAUDE.md 中添加”Compact Instructions”部分或使用焦点运行 /compact（如 /compact focus on the API changes）。

运行 /context 以查看什么在占用空间。MCP 工具定义默认被延迟，并通过工具搜索按需加载，因此只有工具名称消耗上下文，直到 Claude 使用特定工具。运行 /mcp 以检查每个服务器的成本。

我们知道上下文空间是很重要的
上下文空间占用太多的时候 coding成果质量将会大大下降
在上下文窗口即将满的时候 cloude将会做两件事情：
1. 删除以前的工具输出
2. 总结对话 也就是压缩上下文 这一个步骤能很大程度的减轻占用

但是我们也可以手动的进行压缩：使用/compact命令 /compact
并且我们乐意带上描述：制定需要保留 或者说是焦点的内容   /compact focus on the API changes

或者我们可以使用/context命令来看看是什么在占用空间

/mcp 查看 MCP 工具成本 检查哪些工具占用空间多 /memory 查看 Auto Memory 检查跨会话记忆内容
