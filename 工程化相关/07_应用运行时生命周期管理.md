# 07_应用运行时生命周期管理 (App Runtime Lifecycle)

在大型前端工程中，应用启动过程往往涉及数十个初始化任务（如日志上报、参数拉取、权限检查、SDK 加载等）。如果不加管理，这些任务会相互竞争主线程资源，导致首屏卡顿。

本笔记记录了一种**插件化的运行时生命周期调度方案**，旨在通过精细化控制提升启动性能。

---

## 1. 核心生命周期定义 (Lifecycle Stages)

方案将应用启动抽象为四个关键阶段，将技术层面的“加载”转化为业务层面的“状态”。

| 阶段 | 业务语义 | 典型任务 |
| :--- | :--- | :--- |
| **1. init** | 运行环境准备 | 基础参数解析、日志 SDK 初始化、环境配置检查。 |
| **2. visible** | 核心内容可见 | 多语言资源拉取、首屏核心业务配置、UI 骨架屏展示。 |
| **3. interactive** | 交互就绪 | 登录态校验、权限系统初始化、重型功能模块加载。 |
| **4. finish** | 完全完成 | 资源预取 (Prefetch)、数据缓存预热、非核心插件激活。 |

---

## 2. 调度实现机制 (Implementation Mechanism)

### 2.1 任务分类与驱动方式
任务根据其执行性质分为两类，通过不同的方式接入生命周期：

- **命令式任务 (IIFE Tasks)**：
  在特定生命周期触发时直接执行的异步函数。
- **声明式 Hook 任务 (Hook Tasks)**：
  包装成 React Hook 或空组件的任务，挂载到 React 树中利用组件生命周期驱动。

### 2.2 React 组件驱动模型
**Q: 为什么不直接用 `Promise.then` 驱动，而要结合 React？**
**A**: 因为很多业务逻辑（如弹窗检查、主题设置）必须依赖 React Context 或组件状态。通过 React 驱动可以实现“视图与逻辑同步就绪”。

```ascii
[ App Start ]
      |
      v
+-------------+
| Stage: init | (Native JS Execution)
+-------------+
      |
      v
+-----------------------+
| React Root Rendered   |
| <LifecycleConnector>  |
+-----------------------+
      |
      |---- [ visible ] ----> (Triggered by useEffect)
      |
      |---- [ interactive ] --> (Triggered after idle/delay)
      |
      |---- [ finish ] ------> (Triggered after all ready)
```

### 2.3 跨插件依赖管理
插件之间可能存在严格的依赖顺序（例如：上报插件必须等待参数插件就绪）。
- **Ready Promise**: 每个插件实例暴露一个 `readyPromise`。
- **依赖声明**: 在插件配置中声明 `deps: ['params_plugin']`，调度器会自动处理等待逻辑。

---

## 3. 性能优化设计模式 (Optimization Patterns)

### 3.1 性能分层 (Performance Tiering)
**核心思路**：不要在 `main.js` 入口处 `await` 所有东西。
- **非阻塞初始化**：将任务标记为“非关键 (Non-critical)”，允许它们在后台异步执行，而不阻塞 `visible` 阶段的到达。
- **超时兜底**：为每个阶段设置严格的超时阈值（如 `visible` 需在 10s 内到达），超时后强制进入下一阶段或触发错误上报。

### 3.2 业务语义化
**Q: 这种方案和传统的 `DOMContentLoaded` 或 `window.onload` 有什么区别？**
**A**:
- 传统指标是**技术指标**，反映的是浏览器处理 HTML/资源的状态。
- 生命周期阶段是**业务指标**，反映的是“用户什么时候能看到业务数据”以及“什么时候能开始操作”。

---

## 4. 总结 (Summary)

- **统一调度**：将零散的初始化代码集中到插件系统中管理，解决“入口文件臃肿”问题。
- **分阶段加载**：通过生命周期分层，将主线程资源优先分配给 `init` 和 `visible` 阶段，显著优化 TTV (Time to Visible) 指标。
- **可观测性**：每个阶段的耗时、每个插件的加载情况都可被自动统计，为性能优化提供数据支撑。
