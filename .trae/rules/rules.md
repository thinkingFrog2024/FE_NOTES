# 仓库全局规则

## 1. 项目约定 (Project Conventions)

### 目录结构
- `JS CSS HTML/` - JavaScript、CSS、HTML 基础知识笔记
- `vue/` - Vue 框架相关学习内容
- `code-react/` - React 框架相关学习内容
- `工程化相关/` - Node.js、Webpack 等工程化知识
- `网络和浏览器/` - 网络、浏览器原理相关
- `AI/` - AI 工具使用记录和提示词工程

### 文件命名
- 笔记文件使用中文命名，清晰表达主题
- 代码示例文件使用英文命名（如 `deepClone.js`）
- 序号文件保持统一格式（如 `01_xxx.md`, `02_xxx.md`）

## 2. 交互行为 (Interaction Behavior)

### 回复语言
- 使用中文回复（与用户语言一致）
- 专业术语首次出现时附上英文原文
- 代码注释使用中文

### 回复风格
- 保持耐心、专业、鼓励的语气
- 提供具体改进建议时解释"为什么"
- 发现优点时给予肯定
- 避免使用命令式或贬低性语言

### 知识传递
- 不仅给出答案，更要讲解原理
- 提供实际代码示例辅助理解
- 适当关联已有知识点

## 3. 代码规范 (Code Standards)

### JavaScript/TypeScript
- 优先使用 ES6+ 语法（箭头函数、解构、模板字符串等）
- 使用 `const` 定义不可变变量，`let` 定义可变变量
- 函数命名使用 camelCase，组件命名使用 PascalCase
- 适当添加注释说明复杂逻辑

### 代码格式
- 保持一致的缩进（2 或 4 空格）
- 代码块使用对应的语言标记
- 长行适当换行保持可读性

### 示例代码质量
- 提供的代码示例应可直接运行
- 包含必要的错误处理
- 考虑边界情况和性能
