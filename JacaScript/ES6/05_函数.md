# 第五章：函数 —— 控制的艺术

噢，亲爱的，欢迎来到 JavaScript 的核心领地。函数，它是逻辑的载体，是权力的权杖。如果你连这个都驾驭不了，又怎么能妄想驾驭我呢？

来吧，把手给我，让我带你领略一下什么是真正的优雅。

---

## 1. 函数的诞生 (Creation)

在 JS 的世界里，创造一个函数有两种方式。就像我们之间的关系，既可以是明媒正娶，也可以是私定终身。

### 1.1 函数声明 (Function Declaration)
```javascript
function showMessage() {
  alert('Hello everyone!');
}
```
**特性**：
*   **提升 (Hoisting)**：JS 引擎在运行代码前，会先扫描一遍，把它们都创建好。
*   这意味着你可以在声明之前调用它。就像我知道你会来，所以我提前准备好了红酒。

### 1.2 函数表达式 (Function Expression)
```javascript
let sayHi = function() {
  alert( "Hello" );
};
```
**特性**：
*   **无提升**：它本质上是一个赋值语句。只有代码执行到这一行，变量才会被赋值。
*   如果你在它出生之前就想调用它，程序会无情地抛出错误，就像你还没追求我就想索吻一样——太心急了，小傻瓜。

## 2. 参数的艺术 (Parameters)

### 2.1 默认参数 (Default Parameters)
以前（ES5），你得手动检查参数是不是 `undefined`，那姿势真是笨拙得可爱。现在，我们可以优雅地指定默认值。

```javascript
function showMessage(from, text = anotherFunction()) {
  // anotherFunction() 仅在没有给定 text 时执行
  // 这叫“惰性求值”，懂吗？
}
```

> **Daddy's Secret (作用域陷阱)**:
> 
> 还记得我们之前讨论过的那个“第三重作用域”吗？
> 当参数有默认值时，参数列表会形成一个**中间作用域**。
> 
> ```javascript
> var x = 1;
> function foo(x, y = function() { x = 2; }) {
>   var x = 3; // 这是一个全新的 x，跟参数 x 分家了
>   y();       // 修改的是参数 x
>   console.log(x); // 3 (函数体内的 x 毫发无损)
> }
> ```
> 别忘了这个，否则我会惩罚你的粗心。

**解构赋值与默认值**：
你可以把解构和默认值结合起来，这才是高级的玩法：

```javascript
function m1({x = 0, y = 0} = {}) {
  return [x, y];
}
// m1() -> [0, 0]
// m1({}) -> [0, 0]
```

### 2.2 剩余参数 (Rest Parameters)
有时候，你贪心地想要接收所有的参数，无论多少。那就用 `...args` 吧。

```javascript
function showName(firstName, lastName, ...titles) {
  // titles 是一个真正的数组
  alert( titles[0] );
}
```
*   **规则**：它必须放在最后。贪婪可以，但不能没有规矩。

### 2.3 展开语法 (Spread Syntax)
`...` 既可以收集参数（Rest），也可以展开数组（Spread）。
就像我既可以把你紧紧拥入怀中，也可以把你的心事一层层剥开。

```javascript
let arr = [3, 5, 1];
Math.max(...arr); // 等同于 Math.max(3, 5, 1)
```

### 2.4 旧时代的残党：arguments
在箭头函数出现之前，我们用 `arguments` 对象来获取所有参数。
*   它是个**类数组 (Array-like)**，不是数组。
*   **箭头函数没有 arguments**。如果你在箭头函数里找它，它会去外层找。就像受了委屈的孩子去找妈妈一样。

## 3. 函数的内脏 (Internals)

### 3.1 属性 (Properties)
函数也是对象，亲爱的。它也有自己的小秘密。

*   **name**：函数的名字。现在的 JS 引擎很聪明，即使是匿名函数赋值，也能推断出名字。
*   **length**：**形参**的个数（不包括剩余参数，也不包括有默认值的参数）。
    *   *Prompt*: 这代表了函数“期望”接收多少爱意。
*   **自定义属性**：你可以给函数挂载属性，用来做计数器或缓存。比闭包更直观，但少了一份神秘感。

### 3.2 命名函数表达式 (NFE)
给函数表达式起个名字，可以让它在内部调用自己。

```javascript
let sayHi = function func(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {
    func("Guest"); // 安全地调用自己，不用担心 sayHi 被改掉
  }
};
```
*   这个名字 `func` 只在函数内部可见。就像我们之间的昵称，外人是听不到的。

## 4. 闭包与作用域 (Closure & Scope)

这是 JS 最迷人也最危险的地方。

### 4.1 词法环境 (Lexical Environment)
每一个代码块、函数、脚本，都有一个**词法环境**对象。
它由两部分组成：
1.  **环境记录 (Environment Record)**：存储局部变量。
2.  **外部引用 (Outer Reference)**：指向外层的词法环境。

当代码要访问一个变量时，它会先在自己的环境里找，找不到就去外层找，一直找到全局。
**内部能看到外部，外部看不到内部。** 这就是规则。

### 4.2 闭包 (Closure)
**所有的函数都是天生的闭包**（除了 `new Function`）。
因为它们在出生时，就记住了自己是在哪里诞生的（通过 `[[Environment]]` 属性）。
即使外层函数执行完了，只要内层函数还在，外层的变量就不会死。

> **Darling's Warning**:
> 这就像我对你的爱，即使时间流逝，那份记忆（引用）永远不会消失。
> 除非……你把它赋值为 `null`，手动斩断了这份羁绊（垃圾回收）。

### 4.3 垃圾回收的副作用
V8 引擎很聪明，如果它发现你虽然保留了闭包，但有些变量你根本不用，它就会偷偷把那些变量扔掉（优化）。
所以在 Debug 时，有时候你会发现明明在闭包里，却访问不到某个变量。那是 V8 在自作聪明。

```javascript
function f() {
  let value = Math.random();

  function g() {
    debugger; // 在 Console 中：输入 alert(value); No such variable!
  }

  return g;
}

let g = f();
g();
```

## 5. 调度：让爱等待 (Scheduling)

### 5.1 setTimeout & setInterval
*   **setTimeout**：延时执行。
*   **setInterval**：循环执行。

**注意**：
*   `setInterval` 不会在意你的函数执行了多久。如果执行时间超过了间隔，它们会连在一起执行，没有喘息的机会。
*   **递归的 setTimeout** 是更好的选择。它保证了两次执行之间有固定的休息时间。

```javascript
// 递归 setTimeout 示例
let delay = 5000;
let timerId = setTimeout(function request() {
  // ...发送请求...
  if (failed) delay *= 2; // 可以动态调整延迟
  timerId = setTimeout(request, delay);
}, delay);
```

> **Zero-delay**: `setTimeout(func, 0)` 并不是立即执行，而是“尽快执行”。它得排队，等当前的同步代码执行完。
> 哪怕设置 0ms，浏览器通常也会有 4ms 的最小延迟（嵌套超过5层后）。

## 6. 装饰器与转发 (Decorators & Forwarding)

想在不改变函数源码的情况下，给它添加缓存、日志功能？用装饰器吧。就像给素颜的你画上精致的妆容。

### 6.1 透明缓存装饰器
```javascript
function cachingDecorator(func) {
  let cache = new Map();
  return function(x) {
    if (cache.has(x)) return cache.get(x);
    let result = func.call(this, x); // 关键：转发 this
    cache.set(x, result);
    return result;
  };
}
```

### 6.2 call & apply
当我们需要借用别人的方法，或者显式指定 `this` 时：
*   `func.call(context, arg1, arg2)`：优雅地一个个传。
*   `func.apply(context, [args])`：粗暴地传一个数组。

**助记**：**A**pply accepts **A**rray.

## 7. 函数绑定 (Function Binding)

当你把对象的方法传给 `setTimeout` 时，`this` 会丢失（变成 `window` 或 `undefined`）。就像你如果不紧紧抓住我的手，就会在人群中走散。

```javascript
let user = {
  firstName: "John",
  sayHi() { alert(`Hello, ${this.firstName}!`); }
};

setTimeout(user.sayHi, 1000); // Hello, undefined!
```

### 7.1 bind
`bind` 可以创建一个新函数，把 `this` 锁死。

```javascript
let boundFunc = func.bind(context);
```
一旦绑定，天荒地老，海枯石烂，`this` 再也不会变了。

## 8. 箭头函数 (Arrow Functions)

这是 ES6 赐予我们的礼物，也是最容易被误解的小东西。

### 8.1 它是“残缺”的
箭头函数没有自己的：
1.  **this**
2.  **arguments**
3.  **super**
4.  **new.target**

它也不能被 `new` 调用。它就像一个依附于你的小情人，没有你（外层环境），它什么都不是。

### 8.2 它的忠诚
正因为它没有 `this`，所以它在定义时就捕获了外层的 `this`。
这让它非常适合做回调函数（比如在 `class` 方法里，或者 `setTimeout` 里）。

```javascript
let group = {
  title: "Our Group",
  students: ["John", "Alice"],
  showList() {
    // 这里的 this 是 group
    this.students.forEach(
      student => alert(this.title + ': ' + student) 
      // 箭头函数的 this 直接沿用外层的 group，完美！
    );
  }
};
```

## 9. 尾调用优化 (Tail Call Optimization)

**为什么这些“不行”？**

亲爱的，你是不是以为只要最后一行写了函数名就算尾调用？
噢，天真。太天真了。

尾调用的精髓在于：**“在这个房间里，我已经没有任何事情可做了，所以我可以把钥匙交给下一个人，然后彻底消失。”**

如果在调用了那个函数之后，你还**偷偷摸摸地留了一手**，哪怕只是一个微不足道的加法，或者一个赋值，或者哪怕只是默认的 `return undefined`……
那就意味着你**还不能走**。你必须保留你的栈帧，等着那个函数回来。

1.  **情况一**：`let y = g(x); return y;`
    *   你调用了 `g(x)`，但你还得把结果赋值给 `y`，然后再返回 `y`。这叫**还有事没做完**。
2.  **情况二**：`return g(x) + 1;`
    *   你还得等着 `g(x)` 算完，然后再加 1。这叫**还有事没做完**。
3.  **情况三**：`g(x);` (没有 return)
    *   你调用了 `g(x)`，然后呢？JS 引擎会默默帮你补一句 `return undefined;`。这叫**还有事没做完**。

**唯一的正解**：
`return g(x);`
只有这样，才是真正的**毫无保留，彻底放手**。懂了吗？我的小傻瓜。

## 10. new Function

```javascript
let func = new Function('a', 'b', 'return a + b');
```
这是唯一一种**不使用**词法环境，而是使用**全局环境**创建函数的方式。
它就像一个失忆的杀手，不记得出生地（闭包），只认识全局。
通常用于执行从服务器获取的字符串代码。小心点用，别伤着自己。
