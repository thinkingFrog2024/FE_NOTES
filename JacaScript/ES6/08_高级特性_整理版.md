# 高级特性

## 1. Proxy (代理)

Proxy 可以理解为在目标对象之前架设一层“拦截”，外界对该对象的访问，都必须先通过这层拦截。因此提供了一种机制，可以对外界的访问进行过滤和改写。这属于一种**元编程**（Meta Programming），即对编程语言进行编程。

### 1.1 基本用法

```javascript
var proxy = new Proxy(target, handler);
```

- `target`: 目标对象。
- `handler`: 配置对象，用来定制拦截行为。

### 1.2 常用拦截方法 (Traps)

Proxy 支持 13 种拦截操作：

1.  **get(target, propKey, receiver)**: 拦截对象属性的读取。
2.  **set(target, propKey, value, receiver)**: 拦截对象属性的设置。返回 `true` 代表成功。
3.  **has(target, propKey)**: 拦截 `propKey in proxy` 的操作。
4.  **deleteProperty(target, propKey)**: 拦截 `delete proxy[propKey]`。
5.  **ownKeys(target)**: 拦截 `Object.getOwnPropertyNames(proxy)`, `Object.getOwnPropertySymbols(proxy)`, `Object.keys(proxy)`, `for...in` 循环。
6.  **getOwnPropertyDescriptor(target, propKey)**: 拦截 `Object.getOwnPropertyDescriptor(proxy, propKey)`。
7.  **defineProperty(target, propKey, propDesc)**: 拦截 `Object.defineProperty(proxy, propKey, propDesc)`, `Object.defineProperties(proxy, propDescs)`。
8.  **preventExtensions(target)**: 拦截 `Object.preventExtensions(proxy)`。
9.  **getPrototypeOf(target)**: 拦截 `Object.getPrototypeOf(proxy)`。
10. **isExtensible(target)**: 拦截 `Object.isExtensible(proxy)`。
11. **setPrototypeOf(target, proto)**: 拦截 `Object.setPrototypeOf(proxy, proto)`。
12. **apply(target, object, args)**: 拦截 Proxy 实例作为函数调用的操作，比如 `proxy(...)`, `proxy.call(...)`, `proxy.apply(...)`。
13. **construct(target, args)**: 拦截 Proxy 实例作为构造函数调用的操作，比如 `new proxy(...)`。

### 1.3 重点拦截器详解

#### get
- 参数：目标对象、属性名、Proxy 实例本身（receiver）。
- 作用：拦截读取操作。
- 注意：如果一个属性不可配置（configurable）且不可写（writable），则 Proxy 不能修改该属性，否则通过 Proxy 访问该属性会报错。

#### set
- 参数：目标对象、属性名、属性值、Proxy 实例本身（receiver）。
- 作用：拦截赋值操作。
- 注意：严格模式下，`set` 代理如果没有返回 `true`，就会报错。

#### apply
- 参数：目标对象、目标对象的上下文对象（this）、目标对象的参数数组。
- 作用：拦截函数的调用、`call` 和 `apply` 操作。

#### has
- 作用：拦截 `in` 运算符。
- 注意：`has` 拦截对 `for...in` 循环不生效。

#### construct
- 作用：拦截 `new` 命令。
- 注意：必须返回一个对象，否则会报错。

### 1.4 Reflect 对象
`Reflect` 对象与 `Proxy` 对象一样，也是 ES6 为了操作对象而提供的新 API。
- **返回值**: `Reflect` 方法操作失败会返回 `false`，而不是抛出错误（例如写入不可写属性）。
- **配合 Proxy**: 在 Proxy 的拦截器中，通常会调用对应的 `Reflect` 方法来完成默认行为。

### 1.5 `this` 问题
Proxy 代理目标对象时，目标对象内部的 `this` 关键字会指向 Proxy 代理。
- 这可能导致某些依赖 `this` 的原生对象（如 `Date`, `Map`, `Set`, `WeakMap` 等）无法正常工作。
- 解决方案：将方法绑定到原始对象上。

### 1.6 Proxy.revocable()
返回一个可取消的 Proxy 实例。
```javascript
let {proxy, revoke} = Proxy.revocable(target, handler);
proxy.foo = 123;
revoke(); // 取消代理
proxy.foo; // TypeError: Revoked
```

### 1.7 应用场景
1.  **数据验证**: 在 `set` 中检查值是否合法。
2.  **数据绑定/响应式**: Vue 3.0 的核心，追踪依赖，批量更新。
3.  **不可变数据**: 类似于 Immutable.js。
4.  **惰性加载**: 访问属性时才进行计算或加载。
5.  **私有属性保护**: 拦截 `get`/`set`/`ownKeys` 隐藏以 `_` 开头的属性。
6.  **日志监控**: 记录操作日志。

## 2. Decorator (装饰器)

装饰器（Decorator）是一种与类（class）相关的语法，用来注释或修改类和类方法。装饰器是一种函数，写成 `@ + 函数名`。

### 2.1 类的装饰
装饰器可以修改类的行为。
```javascript
@testable
class MyTestableClass {
  // ...
}

function testable(target) {
  target.isTestable = true;
}
```
- 装饰器函数接受的第一个参数就是所要装饰的目标类。
- 如果需要参数，可以在外层再包一层函数。

### 2.2 方法的装饰
装饰器也可以用于类的属性和方法。
```javascript
class Person {
  @readonly
  name() { return `${this.first} ${this.last}` }
}

function readonly(target, name, descriptor) {
  descriptor.writable = false;
  return descriptor;
}
```
- 参数：
  1. `target`: 类的原型对象（Decorator 装饰静态方法时是类本身）。
  2. `name`: 属性名。
  3. `descriptor`: 该属性的描述对象。

### 2.3 执行顺序
- 同一个方法有多个装饰器，会像剥洋葱一样，先从外到内进入，然后由内向外执行。

### 2.4 不能用于函数
装饰器只能用于类和类的方法，不能用于函数，因为存在函数提升。

### 2.5 Mixin 模式
利用装饰器实现 Mixin（混入），将多个类的功能“混入”到一个类中。

### 2.6 新版装饰器提案 (Stage 3)
新版装饰器更加强大，支持 `accessor` 关键字等。
- **Class Decorator**: 接收 `value` (类) 和 `context`。
- **Method Decorator**: 接收 `value` (方法) 和 `context`。
- **Accessor Decorator**: 配合 `accessor` 关键字使用，拦截 `get` 和 `set`。
- **Field Decorator**: 拦截类的实例属性。

## 3. Module (模块)

ES6 模块的设计思想是尽量的静态化，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。

### 3.1 严格模式
ES6 模块自动开启严格模式 (`use strict`)。
- 变量必须声明后再使用。
- 禁止 `this` 指向全局对象（顶层的 `this` 指向 `undefined`）。

### 3.2 export 命令
用于规定模块的对外接口。
- **命名导出**: `export var m = 1;` 或 `export { m };`。
- **默认导出**: `export default function() { ... }`。
- **重命名**: `export { n as m };`。
- **注意**: `export` 输出的是接口，与模块内部的变量是**动态绑定**关系（引用），而非值的缓存。

### 3.3 import 命令
用于输入其他模块提供的功能。
- **基本用法**: `import { m } from './module';`。
- **重命名**: `import { m as n } from './module';`。
- **整体加载**: `import * as circle from './circle';`。
- **只读特性**: `import` 输入的变量通常是只读的（对象属性除外，但不建议修改）。
- **提升**: `import` 命令具有提升效果，会提升到整个模块的头部，首先执行。

### 3.4 export default
- 为模块指定默认输出。
- 导入时不需要大括号: `import customName from './module';`。
- 本质上是导出一个名为 `default` 的变量或方法。

### 3.5 import() 动态加载
ES2020 引入 `import()` 函数，支持动态加载模块。
- 返回一个 Promise 对象。
- 可以在运行时根据条件加载模块。
- 适用于按需加载、条件加载等场景。

### 3.6 import.meta
返回当前模块的元信息。
- `import.meta.url`: 模块的 URL。

### 3.7 ES6 模块与 CommonJS 模块的差异
1.  **CommonJS 模块输出的是一个值的拷贝，ES6 模块输出的是值的引用**。
    - CommonJS 一旦输出，模块内部的变化就影响不到这个值。
    - ES6 是动态引用，JS 引擎对脚本静态分析的时候，遇到模块加载命令 `import`，就会生成一个只读引用。
2.  **CommonJS 模块是运行时加载，ES6 模块是编译时输出接口**。
    - CommonJS 加载的是一个对象（即 `module.exports` 属性）。
    - ES6 模块不是对象，它的对外接口只是一种静态定义，在代码静态解析阶段就会生成。
