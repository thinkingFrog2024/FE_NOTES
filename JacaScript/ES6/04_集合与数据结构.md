# 集合与数据结构
## 数组类型

数组的本质还是对象，所以这和其他基本类型不太一样，数组是可以使用new Array（）声明的。

```javascript
let arr = new Array();
let arr = [];
```

如果使用构造函数创建数字，我们可以传进去数组元素，或者长度，但是如果传递长度的话，每个元素都会是undefined。





**新特性at**

我们可以计算最后一个元素的下标来找到最后一个元素 但是这样写有点麻烦，所以增加了`arr.at(-1)`这种方法来获取最后的元素。这种方法如果i为正数 就和直接取出来一样，如果为负数就是从最后一位往前数。


**pop/push，shift/unshift**

这几种方法都会修改源数组

- `push` 在末端添加一个或几个元素.
- `shift` 取出队列首端的一个元素，整个队列往前移，这样原先排第二的元素现在排在了第一。
- `pop` 从末端取出一个元素并返回 
- `unshift`向队列首端添加一个或几个元素，整个队列往后移。

数组可以作为数据结构：栈 使用 可以通过push/pop操作末端元素。也可以作为队列，在首端取出，在末端添加。

不过，对数组末端操作会比操作首端更快，这是因为操作首端元素的时候后面元素也会受影响。





**数组内部**

正如前面所说的，数组是一种特殊的对象，对对象进行了拓展，使用[]和下标访问数组元素其实源于访问对象属性。

数组和对象一样，复制数组的时候复制的也是数组的引用。

但是**数组的内部实现不一样，js引擎会把这些元素一个一个的存储在连续的内存区域，进行优化，这能使数组运行的很快。但是如果把数组当成对象使用，那么js引擎会关闭对应的优化**。

```javascript
let fruits = []; // 创建一个数组

fruits[99999] = 5; // 分配索引远大于数组长度的属性

fruits.age = 25; // 创建一个具有任意名称的属性
```

数组误用的几种方式:

- 添加一个非数字的属性，比如 `arr.test = 5`。
- 制造空洞，比如：添加 `arr[0]`，然后添加 `arr[1000]` (它们中间什么都没有)。
- 以倒序填充数组，比如 `arr[1000]`，`arr[999]` 等等。



**数组循环**

一般会使用for循环，不过数组也可以使用for of，这种方式无法获取当前的下标。

但是因为数组的本质就是对象，所以其实**for in循环也是可以的，但是这种遍历会遍历所有的属性**，而不仅仅是数字属性。而且这种循环用于数组的时候比较慢。



**length**

这个属性有趣的地方是**他是可写的，增加他不会发生什么，但是减少他数组将会被截断**，并且不可逆。所以如果想要**清空数组，设置长度为零。**



**toString**

数组有自己的toString方法实现，会返回以逗号隔开的元素数组。



**splice**

这个方法可以实现增加 删除 替换：

`arr.splice(start,deleteCount,item1....)`





其他的数组方法：

1. `arr.splice(stsrt[,deletenum,elem1.....elemn])`：这个方法可以删除，添加，插入元素。其实`delete arr[0]`也可以删除，但是这样只会清空某个下标上的内容，length不改变。允许负数索引，表示从尾端开始某位的前面的位置。

2. `arr.slice([start],[end])`复制从start到end（不包括）的元素到新数组，返回新数组，也可以不传参，获得数组的副本。

3. `arr.concat(arg1...argN)`参数可以接收数组或者单个的元素，返回创建的新数组。（类数组按照单个元素处理 但是如果类数组打开了`Symbol.isCincatSpreadable`属性 就会当作数组处理）

   ```javascript
   let arrayLike = {
     0: "something",
     1: "else",

     length: 2
   };

   ```

   

4. `arr.forEach(function(item,index,array){})`对每个数组元素执行特定的操作，可以不一定是函数，alert这样的也可以。如果是函数，那么函数的返回值会被忽略。

5. `arr.indexOf(item[,from])`从from开始寻找item，找到了返回索引，找不到返回-1。这个函数内部使用的是===，无法正确处理NaN

6. `arr.lastIndexOf(item)`和indexOf相同，只不过这个从右向左找。

7. `arr.includes(item[,from])`从from开始查找，找得到返回true，否则返回false。这个函数内部使用的是===，但是他可以正确处理NaN。

8. `arr.find(function(item,index,array){...})` （当然箭头函数也可以）函数内部返回true，则搜索停止，返回对应的item。

9. `arr.findIndex((item,index,array)=>{})`和find一样的，只不过这个返回索引。

10. `arr.findLastIndex(..)` 从右向左 返回索引

11. `arr.filter((item,index,arr)=>{...})` 和find相似，返回所有符合条件的元素组合成的数组。

12. `arr.map((item,index,arr)=>{...})` 对每个元素进行处理 返回处理之后的源数组

13. `arr.sort()` 这个方法默认按照字符串顺序进行排序 如果我们要使用自己定义的比较规则，需要传一个比较器函数。

    ```javascript
    [1, -2, 15, 2, 0, 8].sort( (a, b) => a - b);
    alert( countries.sort( (a, b) => a.localeCompare(b) ) ); // Andorra,Österreich,Vietnam（对的！）
    ```

14. `arr.reverse()` 原地颠倒数组。
15. `arr.split(delim[,length])` 通过给定的分隔符把字符串分隔成数组。第二个参数用于限制数组长度 如果不给定分隔符，会分割成单个的字符。
16. `arr.join(delim)` 返回由给定分隔符分隔的字符串。
17. `arr.reduce(function(accum,item,index,arr){...},[initial])`  用于累积计算。`initial` 是可选的初始值。
    - **如果有 initial**：`accum` 的初始值就是 `initial`，遍历从数组的第一个元素开始（index=0）。
    - **如果没有 initial**：`accum` 的初始值是数组的第一个元素，遍历从数组的第二个元素开始（index=1）。
    - 每次函数执行的返回值，会作为下一次执行的 `accum` 参数。
18. `arr.reduceRight(function(accum,item,index,arr){...},[initial])` 和上面哪个一样 但是从右边开始迭代
19. `Array.isArray(arr)` 因为数组是基于对象的 所以typeof会得到object，这种方式可以区分数组和对象 返回true/false



1. `arr.some(fn)` `arr.every(fn)`

2. `arr.fill(val,start,end)`

3. `arr.copyWithin(target, start, end)`  **在当前数组内部**，把 `[start, end)` 区间内的元素复制一份，然后粘贴到 `target` 开始的位置。
    - **这个方法会修改原数组**，但不会改变数组的长度。
    - `target`：粘贴的目标起始位置（如果是负数，从倒数开始算）。
    - `start`：复制的起始位置（默认 0）。
    - `end`：复制的结束位置（默认到数组末尾，不包含该位置）。
    - *例子：`[1, 2, 3, 4, 5].copyWithin(0, 3)` 意思是把从索引 3 开始的元素 `[4, 5]` 复制，粘贴到索引 0 的位置，结果变成 `[4, 5, 3, 4, 5]`*。

4. `arr.flat(depth)` 与 `arr.flatMap(fn)`
    - `flat(depth)`：纯粹的“拍平”数组。
      - `depth` 是拍平的深度，默认为 1。如果想无限拍平，可以传 `Infinity`。
      - *例子：`[1, [2, [3]]].flat(2)` -> `[1, 2, 3]`*
    - `flatMap(fn)`：先执行 `map(fn)`，然后对结果执行 `flat(1)`。
      - **效率更高**：因为它只遍历一次。
      - **只能拍平一层**。
      - **妙用**：可以同时实现 `map` 和 `filter` 的功能（如果不想要某个元素，返回空数组 `[]`，拍平后就消失了；如果想要，返回 `[value]`，拍平后就留下了）。

   1. 由于之前的很多方法都是直接修改原数组，现在有了一些返回拷贝数组的方法：`toReversed()` `toSorted()`  `toSpliced()`   `with(index,value)`

   ```javascript
   const sequence = [1, 2, 3];
   sequence.toReversed() // [3, 2, 1]
   sequence // [1, 2, 3]
   
   const outOfOrder = [3, 1, 2];
   outOfOrder.toSorted() // [1, 2, 3]
   outOfOrder // [3, 1, 2]
   
   const array = [1, 2, 3, 4];
   array.toSpliced(1, 2, 5, 6, 7) // [1, 5, 6, 7, 4]
   array // [1, 2, 3, 4]
   
   const correctionNeeded = [1, 1, 3];
   correctionNeeded.with(1, 2) // [1, 2, 3]
   correctionNeeded // [1, 1, 3]
   ```

   


**`Object.groupBy()` 和 `Map.groupBy()`**

（注意：之前这两个方法是作为数组实例方法 `arr.group()` 提案的，但最终标准定为了静态方法 `Object.groupBy()` 和 `Map.groupBy()`）

这两个方法接收一个可迭代对象（如数组）和一个分组函数，根据函数的返回值将元素分组。

1.  **`Object.groupBy(items, callback)`**
    - 返回一个**普通对象**。
    - 分组的键会被强制转换为字符串（或 Symbol）。
    - 适合简单的字符串键分组。

    ```javascript
    const inventory = [
      { name: "asparagus", type: "vegetables", quantity: 5 },
      { name: "bananas", type: "fruit", quantity: 0 },
      { name: "goat", type: "meat", quantity: 23 },
      { name: "cherries", type: "fruit", quantity: 5 },
      { name: "fish", type: "meat", quantity: 22 },
    ];

    const result = Object.groupBy(inventory, ({ type }) => type);
    
    /* 结果：
    {
      vegetables: [{ name: "asparagus", ... }],
      fruit: [{ name: "bananas", ... }, { name: "cherries", ... }],
      meat: [{ name: "goat", ... }, { name: "fish", ... }]
    }
    */
    ```

2.  **`Map.groupBy(items, callback)`**
    - 返回一个 **Map** 对象。
    - 分组的键可以是**任何类型**（对象、布尔值、数字等）。
    - 适合复杂键的分组。

    ```javascript
    // 按库存是否充足分组（键是 boolean）
    const restock = Map.groupBy(inventory, ({ quantity }) => quantity < 6);
    
    console.log(restock.get(true)); 
    // [{ name: "asparagus", ... }, { name: "bananas", ... }, { name: "cherries", ... }]
    ```

**可迭代对象**

可迭代对象是数组的泛化，这个概念的意思是所有对象都可以被定制成能通过for of循环的对象

比如我有一个对象，适合使用for of循环：

```javascript
let range = {
  from: 1,
  to: 5
};

// 我们希望 for..of 这样运行：
// for(let num of range) ... num=1,2,3,4,5
```

为了让 `range` 对象可迭代（也就让 `for..of` 可以运行）我们需要为对象添加一个名为 `Symbol.iterator` 的方法（一个专门用于使对象可迭代的内建 symbol）。

1. 当 `for..of` 循环启动时，它会调用这个方法（如果没找到，就会报错）。这个方法必须返回一个 **迭代器（iterator）** —— 一个有 `next` 方法的对象。
2. 从此开始，`for..of` **仅适用于这个被返回的对象**。
3. 当 `for..of` 循环希望取得下一个数值，它就调用这个对象的 `next()` 方法。
4. `next()` 方法返回的结果的格式必须是 `{done: Boolean, value: any}`，当 `done=true` 时，表示循环结束，否则 `value` 是下一个值。

```javascript
let range = {
  from: 1,
  to: 5
};

// 1. for..of 调用首先会调用这个：
range[Symbol.iterator] = function() {

  // ……它返回迭代器对象（iterator object）：
  // 2. 接下来，for..of 仅与下面的迭代器对象一起工作，要求它提供下一个值
  return {
    current: this.from,
    last: this.to,

    // 3. next() 在 for..of 的每一轮循环迭代中被调用
    next() {
      // 4. 它将会返回 {done:.., value :...} 格式的对象
      if (this.current <= this.last) {
        return { done: false, value: this.current++ };
      } else {
        return { done: true };
      }
    }
  };
};

// 现在它可以运行了！
for (let num of range) {
  alert(num); // 1, 然后是 2, 3, 4, 5
}
//从技术上说，我们可以将它们合并，并使用 range 自身作为迭代器来简化代码。
//现在 range[Symbol.iterator]() 返回的是 range 对象自身：它包括了必需的 next() 方法，并通过 this.current 记忆了当前的迭代进程。这样更短，对吗？是的。有时这样也可以。

//但缺点是，现在不可能同时在对象上运行两个 for..of 循环了：它们将共享迭代状态，因为只有一个迭代器，即对象本身。
//解释：
//正常情况下，`[Symbol.iterator]()` 应该每次返回一个新的对象 `{ next() {...} }`，这个新对象里保存着独立的 `current` 状态。
//但是在这里，为了偷懒，我们让 `[Symbol.iterator]()` 直接返回了 `this`（对象自己）。
//这意味着，迭代的状态 `current` 是保存在对象自己身上的。
//如果你启动了两个循环，它们操作的是同一个对象的同一个 `current` 属性。
//A 循环把 `current` 加到了 2，B 循环再来读的时候，直接就从 2 开始了，这就乱套了。
let range = {
  from: 1,
  to: 5,

  [Symbol.iterator]() {
    this.current = this.from;
    return this;
  },

  next() {
    if (this.current <= this.to) {
      return { done: false, value: this.current++ };
    } else {
      return { done: true };
    }
  }
};

for (let num of range) {
  alert(num); // 1, 然后是 2, 3, 4, 5
}
```

> 无穷迭代器
>
> 无穷迭代器也是可能的。例如，将 `range` 设置为 `range.to = Infinity`，这时 `range` 则成为了无穷迭代器。或者我们可以创建一个可迭代对象，它生成一个无穷伪随机数序列。也是可能的。
>
> `next` 没有什么限制，它可以返回越来越多的值，这是正常的。
>
> 当然，迭代这种对象的 `for..of` 循环将不会停止。但是我们可以通过使用 `break` 来停止它。

字符串也是可以迭代的（这是为什么？因为字符串这种数据类型实现了迭代器协议 这是ES6里引入的一种机制）

**显式调用迭代器**

```javascript
let str = "Hello";

// 和 for..of 做相同的事
// for (let char of str) alert(char);

let iterator = str[Symbol.iterator]();

while (true) {
  let result = iterator.next();
  if (result.done) break;
  alert(result.value); // 一个接一个地输出字符
}
```

可迭代（iterable）数据:实现了Symbol.iterator   

类数组（array-like）对象：具有索引和length属性的对象。就比如字符串，既是可迭代的，又是类数组。

类数组对象通常并不是数组，但是如果我们希望像数组一样操作他们，可以使用`Array.from(obj[,mapFn.thisArg])`方法，接收一个可迭代对象/类数组对象，将其处理成一个真正的数组并返回。`mapFn` 可以是一个函数，该函数会在对象中的元素被添加到数组前，被应用于每个元素，此外 `thisArg` 允许我们为该函数设置 `this`。

```javascript
let arr = Array.from(range, num => num * num);

alert(arr); // 1,4,9,16,25
```

这个函数也可以把字符串转换成单个字符的数组，和split不同 它依赖于字符串的可迭代性，这意味着它会按照字符串的规范来正确处理代理对。（迭代器可以正确处理代理对）

> **什么是代理对 (Surrogate Pair)？**
>
> JavaScript 内部使用 UTF-16 编码，每个字符占 16 位（2字节）。
> 但是，像 Emoji (`😂`) 或一些生僻汉字 (`𩷶`)，它们的码点超出了 16 位所能表示的范围（它们是 32 位的）。
> 为了表示这些字符，JS 会用**两个** 16 位的单元拼在一起，这两个单元合起来就叫**代理对**。
>
> - `str.length` 会把代理对算作 2 个长度。
> - `str.split('')` 会把代理对切开，变成两个乱码字符。
> - `Array.from(str)` 或 `for...of` 会智能识别代理对，把它们当做一个完整的字符处理。
>
> ```javascript
> let str = '𝒳😂'; 
> console.log(str.length); // 4 (每个字符占2个单位)
> console.log(str.split('')); // ['\uD835', '\uDCB3', '\uD83D', '\uDE02'] (乱码，拆散了)
> console.log(Array.from(str)); // ['𝒳', '😂'] (正确，识别了代理对)
> ```

`str.split` 方法在处理字符串时，通常按照字符串的分隔符来分割字符串。默认情况下，如果没有提供分隔符，它会按照字符串的每个字符来分割，但这里的“字符”实际上是按照UTF-16代码单元来分割的。这意味着，如果一个字符由一个代理对表示，`str.split` 方法可能会将这个字符分割成两部分，因为每个代理对代码单元被当作独立的字符。

像原生的slice方法，也无法识别代理对，我们可以通过这个函数改进他

```javascript
function slice(str, start, end) {
  return Array.from(str).slice(start, end).join('');
}

let str = '𝒳😂𩷶';

alert( slice(str, 1, 3) ); // 😂𩷶

// 原生方法不支持识别代理对（译注：UTF-16 扩展字符）
alert( str.slice(1, 3) ); // 乱码（两个不同 UTF-16 扩展字符碎片拼接的结果）
```



**原型方法**

**Array.from()** :这个方法用于把类数组对象 可遍历对象转换成数组

```javascript
let arrayLike = {
    '0': 'a',
    '1': 'b',
    '2': 'c',
    length: 3
};

// ES5 的写法  在一个类数组对象的上下文里面执行slice方法
var arr1 = [].slice.call(arrayLike); // ['a', 'b', 'c']

// ES6 的写法
let arr2 = Array.from(arrayLike); // ['a', 'b', 'c']
```

具有遍历器接口的数据结构都可以使用这个方法。之前提到的使用拓展运算符也可以把某些具有遍历器的数据变成数组，这两个方法的差别在于：

1.  **`Array.from()`**：更强大。它只需要对象有 `length` 属性，就可以把它变成数组。如果对象里没有对应的索引值（比如只有 length=3，但没有 0,1,2 属性），它会把空缺填补为 `undefined`。
2.  **扩展运算符 `...`**：更挑剔。它**必须**要求对象部署了 Iterator 接口（即 `[Symbol.iterator]`）。如果一个类数组对象没有 Iterator 接口，用 `...` 会直接报错。

```javascript
const arrayLike = { length: 3 };

// Array.from 能处理：
Array.from(arrayLike); // [undefined, undefined, undefined]

// 扩展运算符不能处理：
// [...arrayLike]; // 报错：object is not iterable
```

这个方法还可以接受一个函数作为第二个参数,类似于数组的map方法，

另外这个方法由于可以正确处理各种字符 可以使用这个方法把字符串转换成数组，再取数组的长度，避免错误。



**Array.of()**方法用于把一组值转换成数组，可以弥补构造函数的问题：

```javascript

Array(3,33,3)//得到[3,33,3]
//只有当参数不少于两个 构造函数才会使用参数构成新的数组
```

## Map Set

Map是比对象更加完整的哈希结构，允许任何数据类型作为键，键的类型不会被转换成字符串。

它的方法和属性如下：

- `new Map()` —— 创建 map。
- `map.set(key, value)` —— 根据键存储值。
- `map.get(key)` —— 根据键来返回值，如果 `map` 中不存在对应的 `key`，则返回 `undefined`。
- `map.has(key)` —— 如果 `key` 存在则返回 `true`，否则返回 `false`。
- `map.delete(key)` —— 删除指定键的值。
- `map.clear()` —— 清空 map。
- `map.size` —— 返回当前元素个数。

>
> 这种写法虽然**看起来**像是在操作对象，但它实际上并没有把数据存进 Map 的数据结构里，而是给 Map 这个对象实例添加了一个普通的属性。
>
> 1.  **数据并没有真的存进去**：`map.has(key)` 会返回 `false`，`map.size` 也不会增加。
> 2.  **迭代不到**：`for...of` 循环根本看不到这个属性。
> 3.  **键被强制转成了字符串**：Map 本来支持任何类型的键（比如对象），但如果你用 `map[obj] = val`，它会把 `obj` 强转成字符串 `"[object Object]"`，这就失去了 Map 最大的优势。
>
> **记住：既然用了 Map，就老老实实给我用 `.set()` 和 `.get()`。**

> Map是怎么比较键的？
>
> 内部使用的是SameValueZero算法 和===差不多 但是NaN会等于它本身 所以NaN也可以用作键

> 链式调用
>
> 每一次map.set调用之后都会返回map本身，所以可以：map.set().set()....

`WeakMap` 只有以下的方法：

- `weakMap.get(key)`
- `weakMap.set(key, value)`
- `weakMap.delete(key)`
- `weakMap.has(key)`

这是因为如果有一个对象丢失了所有引用，那么按道理，他会被回收，但是我们不能准确的知道这个对象到底在什么时候会被回收。所以不支持访问所有的键值。

这种数据类型的使用场景是额外数据的存储，比如我们正在处理某个对象的一些数据，那么要是这个对象被清除了，和对象有关的数据也应该被一起清除。另一个例子是缓存，我们可以把对象和对象的一些缓存计算数据存储在里面，当用户被清理，缓存数据也会清理。这样的方式，避免手动清除

```javascript

let cache = new WeakMap();

// 计算并记结果

  if (!cache.has(obj)) {
    let result = /* calculate the result for */ obj;

    cache.set(obj, result);
  }

  return cache.get(obj);
}

// 📁 main.js
let obj = {/* some object */};

let result1 = process(obj);
let result2 = process(obj);

// ……稍后，我们不再需要这个对象时：
obj = null;

// 无法获取 cache.size，因为它是一个 WeakMap，
// 要么是 0，或即将变为 0
// 当 obj 被垃圾回收，缓存的数据也会被清除
```

## iterator

Iterator的遍历过程：创建一个指针对象，指向当前数据结构的起始位置 不断调用next方法 知道这个指针指向了数据结构的末尾。每一次调用next方法 都会返回一个包含value done属性的对象。

```javascript
//一个简单的创建遍历器函数
var it = makeIterator(['a', 'b']);

it.next() // { value: "a", done: false }
it.next() // { value: "b", done: false }
it.next() // { value: undefined, done: true }

function makeIterator(array) {
  var nextIndex = 0;
  return {
    next: function() {
      return nextIndex < array.length ?
        {value: array[nextIndex++], done: false} :
        {value: undefined, done: true};
    }
  };
}
//而且done value这两个属性都可以省略
function makeIterator(array) {
  var nextIndex = 0;
  return {
    next: function() {
      return nextIndex < array.length ?
        {value: array[nextIndex++]} :
        {done: true};
    }
  };
}
```

迭代器并不是和数据结构仅仅相连的，遍历器和它所遍历的那个数据结构实际上是分开的，所以遍历器是可以没有对应的数据结构的

```javascript
interface Iterable {
  [Symbol.iterator]() : Iterator,
}

interface Iterator {
  next(value?: any) : IterationResult,
}

interface IterationResult {
  value: any,
  done: boolean,
}
```



ES6 规定，默认的 Iterator 接口部署在数据结构的`Symbol.iterator`属性，或者说，一个数据结构只要具有`Symbol.iterator`属性，就可以认为是“可遍历的”（iterable）。

原生具备 Iterator 接口的数据结构如下。

- Array
- Map
- Set
- String
- TypedArray
- 函数的 arguments 对象
- NodeList 对象

其实给对象部署遍历器很没有必要 用map就可以了。

我们可以给类部署遍历器：

```javascript
class RangeIterator {
  constructor(start, stop) {
    this.value = start;
    this.stop = stop;
  }

  [Symbol.iterator]() { return this; }

  next() {
    var value = this.value;
    if (value < this.stop) {
      this.value++;
      return {done: false, value: value};
    }
    return {done: true, value: undefined};
  }
}

function range(start, stop) {
  return new RangeIterator(start, stop);
}

for (var value of range(0, 3)) {
  console.log(value); // 0, 1, 2
}
```

用遍历器实现链表：

```javascript
function Obj(value) {
  this.value = value;
  this.next = null;
}

Obj.prototype[Symbol.iterator] = function() {
  var iterator = { next: next };

  var current = this;

  function next() {
    if (current) {
      var value = current.value;
      current = current.next;
      return { done: false, value: value };
    }
    return { done: true };
  }
  return iterator;
}

var one = new Obj(1);
var two = new Obj(2);
var three = new Obj(3);

one.next = two;
two.next = three;

for (var i of one){
  console.log(i); // 1, 2, 3
}
```

给类数组对象部署遍历器：

```javascript
NodeList.prototype[Symbol.iterator] = Array.prototype[Symbol.iterator];
// 或者
NodeList.prototype[Symbol.iterator] = [][Symbol.iterator];

[...document.querySelectorAll('div')] // 可以执行了
```

调用遍历器的场景：

1. 解构赋值：对数组和set进行解构赋值将会调用遍历器。
2. 拓展运算符：使用拓展运算符可以把所有具有遍历器的数据变成数组
3. yield*后面跟随可迭代对象时

```javascript
function* generateFromArray(array) {
  yield* array; // 委托给数组
}

const array = [1, 2, 3];
const generator = generateFromArray(array);

console.log(generator.next().value); // 输出：1
console.log(generator.next().value); // 输出：2
console.log(generator.next().value); // 输出：3
console.log(generator.next().done);  // 输出：true，表示迭代完成
```

## 进阶：数据结构与内存管理

听好了，这部分是进阶内容。别只是死记硬背 API，要理解背后的原理。

### 1. 稀疏数组 (Sparse Array) 的性能陷阱

虽然 `arr = []` 看起来很简单，但当你制造“空洞”时，V8 引擎会把数组降级。

```javascript
const arr = [1, 2, 3]; // 密集数组 (Dense Array) - 很快，内存连续
arr[1000] = 4;         // 稀疏数组 (Sparse Array) - 慢，变成了哈希表 (Hash Table)
```

**为什么慢？**
*   密集数组：直接通过偏移量访问内存地址。
*   稀疏数组：需要查找哈希表。

**建议**：
*   永远不要手动制造空洞（不要 `delete arr[i]`，用 `splice`）。
*   初始化大数组时，用 `new Array(n).fill(0)` 把它填满，保持密集。

### 2. Set 的数学运算

Set 不仅仅是去重，它就是数学上的“集合”。ES2024 新增了原生的集合运算方法，不用再手写了。

```javascript
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

// 1. 并集 (Union): a ∪ b
const union = a.union(b); // Set {1, 2, 3, 4}

// 2. 交集 (Intersection): a ∩ b
const intersection = a.intersection(b); // Set {2, 3}

// 3. 差集 (Difference): a - b (a 有但 b 没有的)
const difference = a.difference(b); // Set {1}

// 4. 对称差集 (Symmetric Difference): (a - b) ∪ (b - a)
const symmetricDifference = a.symmetricDifference(b); // Set {1, 4}

// 5. 子集判断 (Is Subset Of)
const isSubset = new Set([1, 2]).isSubsetOf(a); // true
```

*(注意：如果你的环境还不支持这些新方法，记得用 Polyfill 或者自己实现。)*

### 3. WeakRef 和 FinalizationRegistry

这是比 WeakMap 更底层的弱引用控制，专门给那些需要精细管理内存的高手用的。

*   **WeakRef**: 允许你持有一个对象的弱引用，但不阻止它被垃圾回收。
*   **FinalizationRegistry**: 当一个对象被回收时，通知你。

```javascript
let target = { id: 1 };
const weakRef = new WeakRef(target);

const registry = new FinalizationRegistry((heldValue) => {
  console.log(`对象被回收了: ${heldValue}`);
});

registry.register(target, "target对象");

target = null; // 释放强引用

// 过一段时间，GC 运行后...
// console: "对象被回收了: target对象"

// 如果还没回收，可以尝试获取
const deref = weakRef.deref();
if (deref) {
  console.log("对象还在:", deref);
} else {
  console.log("对象没了");
}
```

**警告**：垃圾回收的时机是不可预测的。别在业务逻辑里依赖它，只用来做清理工作（比如清除缓存、断开连接）。

### 4. TypedArray (类型化数组)

如果你在处理二进制数据（比如文件流、WebGL、WebAssembly），普通的 Array 就不够用了。你需要 `TypedArray`。

这东西有点反直觉，因为它把**内存**和**视图**分开了。

#### 核心概念：Buffer 和 View

1.  **ArrayBuffer (内存条)**：
    *   这是一块纯粹的、连续的二进制内存空间。
    *   你**不能直接操作它**（不能读也不能写），它就像一块未开发的荒地。
    *   `new ArrayBuffer(16)`：申请 16 个字节的内存。

2.  **TypedArray / DataView (视图/眼镜)**：
    *   为了操作这块内存，你需要戴上“眼镜”（视图）。
    *   **Uint8Array**：把这块地切成 1字节一份，当成无符号整数（0-255）来看。
    *   **Int32Array**：把这块地切成 4字节一份，当成有符号整数来看。
    *   **关键点：多个视图可以指向同一块内存！**

```javascript
// 1. 申请 16 字节的内存
const buffer = new ArrayBuffer(16); 

// 2. 创建视图
const view1 = new Uint8Array(buffer); // 把内存看作 16 个 1 字节的整数
const view2 = new Int32Array(buffer); // 把内存看作 4 个 4 字节的整数

// 3. 修改 view2 的第一个元素
view2[0] = 123456; 

// 4. view1 也会变！因为它们共用同一块内存
console.log(view1[0]); // 64 (123456 的低 8 位)
```

#### 为什么它快？
1.  **没有“空洞”**：内存是连续的，不像普通数组是哈希表。
2.  **类型单一**：引擎不需要检查类型，直接读写内存地址。

#### 常用视图类型
*   `Int8Array`: 8位有符号整数 (1字节)
*   `Uint8Array`: 8位无符号整数 (1字节，范围 0-255)
*   `Float32Array`: 32位浮点数 (4字节，WebGL 常用)

> **纠正一个常见误区**：
> 1 字节 (Byte) = 8 位 (bit)。
> 也就是说，一个 `Uint8Array` 的元素，刚好占用 1 个字节。
> 而 `Int32Array` 的元素，占用 4 个字节 (32位)。

#### 实战场景：到底哪里用得上？

如果你觉得“没什么用”，那是因为你还没碰到需要高性能处理二进制数据的场景。

1.  **Canvas / WebGL 图像处理**：
    *   一张 4K 图片 (3840x2160) 有 800 万个像素。
    *   每个像素有 R, G, B, A 四个通道（0-255）。
    *   总共需要 32MB 的内存。
    *   如果用普通数组存 3200 万个数字，浏览器直接卡死。用 `Uint8ClampedArray`，瞬间搞定。

2.  **文件上传 / 下载**：
    *   大文件分片上传 (File Slice)。
    *   在前端解析文件头（判断是 PNG 还是 JPG）。

3.  **WebSocket / 实时通信 (Protocol Buffers, MsgPack)**：
    *   **问题**：你习惯用的 JSON (`JSON.stringify`) 是文本格式，非常臃肿。
    *   **例子**：传一个 `{ id: 123456789, name: "Alice" }`。
        *   JSON: `{"id":123456789,"name":"Alice"}` -> **34 字节**。
        *   二进制 (MsgPack/Protobuf): `[Hex] 15 CD 5B 07 05 41 6C 69 63 65` -> **10 字节**。
    *   **差距**：节省了 **70%** 的带宽！在百万级并发的即时通讯（IM）或游戏里，这节省下来的流量就是真金白银。
    *   **怎么做**：后端传给你二进制流 (ArrayBuffer)，你前端用 TypedArray 接收，然后用库解析。
    *   *如果你连这个都没听过，说明你做的项目流量还不够大。*

### 5. 异步迭代器 (Async Iterator)

普通的迭代器是同步的，但如果数据来自网络请求或文件流呢？我们需要异步迭代器。

*   接口：`Symbol.asyncIterator`
*   方法：`next()` 返回 `Promise<{value, done}>`
*   循环：`for await (let item of iterable)`

```javascript
const asyncIterable = {
  [Symbol.asyncIterator]() {
    let i = 0;
    return {
      next() {
        if (i < 3) {
          return Promise.resolve({ value: i++, done: false });
        }
        return Promise.resolve({ done: true });
      }
    };
  }
};

(async () => {
  for await (let num of asyncIterable) {
    console.log(num); // 0, 1, 2
  }
})();
```

### 6. Map vs Object: 真正的高手怎么选？

别无脑用 `Object`，也别无脑用 `Map`。

*   **内存占用**: `Map` 比 `Object` 大约多占 50% 内存（因为要存哈希表结构）。如果你存几百万个小对象，用 `Object` 更省内存。
*   **插入/删除性能**: `Map` 在频繁增删键值对时性能更好（V8 专门优化过）。`Object` 的 `delete` 操作非常慢（会破坏隐藏类）。
*   **键的类型**: 需要非字符串键？必须用 `Map`。
*   **迭代顺序**: `Map` 严格按插入顺序。`Object` 大致按插入顺序，但整数键会排在前面（坑点！）。

**结论**：
*   固定结构的实体（如 `User`, `Config`） -> 用 `Object`。
*   频繁增删、键名不可控、需要非字符串键 -> 用 `Map`。

### 7. Array.fromAsync (ES2024)

`Array.from` 的异步版本。如果你的输入是一个异步迭代器（或者包含 Promise 的数组），它会等待所有 Promise 完成。

```javascript
const asyncItems = [
  Promise.resolve(1),
  Promise.resolve(2),
  Promise.resolve(3),
];

// 老写法：Promise.all
const arr1 = await Promise.all(asyncItems);

// 新写法：Array.fromAsync
const arr2 = await Array.fromAsync(asyncItems);
```

它也支持异步生成器：

```javascript
async function* generate() {
  yield 1;
  yield 2;
  yield 3;
}

const arr = await Array.fromAsync(generate()); // [1, 2, 3]
```

### 8. JSON.stringify 的“坑”

别以为 `JSON.stringify` 万能。

1.  **Map / Set**: 默认会被序列化成空对象 `{}`。
    ```javascript
    const map = new Map([['a', 1]]);
    console.log(JSON.stringify(map)); // "{}" (坑！)
    // 正确做法：先转数组
    console.log(JSON.stringify([...map])); // "[['a',1]]"
    ```
2.  **BigInt**: 直接报错 `TypeError: Do not know how to serialize a BigInt`。
    *   *解决方法*：给 BigInt 原型加个 `toJSON` 方法。
3.  **循环引用**: 直接报错。
4.  **Symbol / Undefined / Function**:
    *   作为对象属性值：被忽略。
    *   作为数组元素：变成 `null`。
