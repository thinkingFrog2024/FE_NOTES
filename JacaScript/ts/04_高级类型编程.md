# 高级类型编程

## 名义类型

我们可以通过名义类型 把对参数的验证逻辑转移到核心逻辑之外：
```typescript
declare const TYPE_TAG: unique symbol; // 2.7+
// 通过 & 并入一个特殊属性来定义名义类型（一些人将这个过程称为 Tagging）
type PositiveValue = number & { [TYPE_TAG]: '_' };

// 我们的 API
declare function doSomething(value: PositiveValue): void;

// 我们自己提供一个将 number 转为 PositiveValue 的函数
function asPositiveValue(value: number): PositiveValue {
  if (value <= 0) {
    throw new Error("必须为正数");
  }
  return value as PositiveValue;
}

doSomething(-123); // 报错
doSomething(asPositiveValue(123)); // 不报错
```
这里我们给一个通过类型检查的变量打上一个特殊的编译时的标记 所以参数必须通过这个函数的检查 并且如果参数不满足要求 将会在运行时抛出一个错误。
但是由于 `type PositiveValue = number & { [TYPE_TAG]: '_' };` 实际上的计算结果是一个 `never` 类型 所以我们在函数里面使用这个变量的时候 需要重新对其进行类型断言。

## 阻止更加简洁的联合类型 (Smarter Type Alias Preservation)

```typescript
type Foo = 1 | 2 | 3;
type Bar = Foo | 4 | 5;
//   ^? type Bar = Foo | 4 | 5
```
自 TypeScript 4.2 版本起，为了提升代码的可读性，编译器引入了一项名为“Smarter Type Alias Preservation”的特性。
TypeScript 认为，在类型提示中显示 `Foo | 4 | 5` 比显示 `1 | 2 | 3 | 4 | 5` 更加简洁明了。如果 Foo 是一个从外部库导入的复杂类型，保留其别名 Foo 会让我们更容易理解 Bar 的结构。

但是有的时候我们认为这个类型提示有一些不直观 如果我们希望得到更加直接的提示 我们可以使用这个小技巧：
```typescript
type Foo = 1 | 2 | 3;
type Bar = Foo & {} | 4 | 5;
//   ^? type Bar = 1 | 2 | 3 | 4 | 5
```
这里的 `& {}` 是一个广为人知的技巧，它的作用是强制 TypeScript 立即（Eagerly）计算出类型的完整形态，从而阻止上述的“别名保留”行为。

为什么 `& {}` 能做到这一点？
编译器的“别名保留”逻辑虽然智能，但它有自己的原则。当它遇到一个必须知道类型内部结构才能继续的操作时，它就不得不放弃“偷懒”，开始真正的计算。交叉类型 `&` 就是这样一个操作。
1. `{}` 类型的真正含义：在 TypeScript 中，`{}` 类型并不单指“空对象”，它代表的是任何非 null 和非 undefined 的值。因此，原始类型 string、number 等都可以被赋值给 `{}` 类型。
2. 交叉运算的结果：正因为如此，一个原始类型与 `{}` 进行交叉运算，结果仍然是它自身。例如，`1 & {}` 的结果就是 1。
3. 触发信号：尽管 `& {}` 在类型上看似是一个“无操作”（no-op），但 `&` 运算符本身的存在，就像一个发给编译器的强制求值信号。它告诉编译器：“别再保留别名了，我需要你立刻展开 Foo，因为我要对它的内部结构进行交叉运算。”

让我们一步步拆解编译器的思考过程：
1. 分析表达式：编译器看到 `Foo & {} | 4 | 5` 这个表达式。根据运算符优先级，它需要先处理 `Foo & {}`。
2. 触发求值：为了计算 `Foo & {}` 的结果，编译器必须知道 Foo 的具体形态。它不能再简单地把 Foo 当作一个黑盒。
3. 展开别名：编译器将 Foo 展开为其原始定义：`1 | 2 | 3`。
4. 执行交叉运算：现在，表达式变成了 `(1 | 2 | 3) & {}`。交叉运算会分配到联合类型的每一个成员上，即 `(1 & {}) | (2 & {}) | (3 & {})`。

并且联合类型还存在 `subtype reduction` 这种情况 例如一个联合类型：`'foo' | string` 在提示的时候 我们看到的结果却是 `string`。
这是因为 `'foo'` 是 `string` 的子类型 所以ts省略了他。
但是我们并不希望这样的省略 于是：
```typescript
declare function foo(input: 'a' | 'b' | string & {}): void;
```
这里的原理（rationale）是通过提供一个名义类型阻止了 TypeScript 的约去动作。（TS 5.4已经解决这个问题了）

## Immediately Indexed Mapped Type（IIMT）

```typescript
export type ResultType = {
// 👇 T 取自联合类型      👇 使用 T 构建对应的对象类型
  [T in SomeUnionType]: {};
}[SomeUnionType]; // 👈 创建一个马上运行的映射类型（mapped type）
```

```typescript
type CSSUnits = "px" | "em" | "rem" | "vw" | "vh";

/**
 * | { length: number; unit: 'px'; }
 * | { length: number; unit: 'em'; }
 * | { length: number; unit: 'rem'; }
 * | { length: number; unit: 'vw'; }
 * | { length: number; unit: 'vh'; }
 */
export type CSSLength = {
  [U in CSSUnits]: {
    length: number;
    unit: U;
  };
}[CSSUnits];
```

```typescript
type Event =
  | { type: "click"; x: number; y: number; }
  | { type: "hover"; element: HTMLElement; };

// 对某个含有 type 属性的对象类型，将它的 type 属性加上一个字符串前缀
// 同时，其它属性保持不变
type PrefixType<E extends { type: string }> = {
  type: `PREFIX_${E["type"]}`;
} & Omit<E, "type">;

/**
 * | { type: 'PREFIX_click'; x: number; y: number; }
 * | { type: 'PREFIX_hover'; element: HTMLElement; }
 */
type Example = {
//               👇 使用了「映射类型中键的重映射」
  [E in Event as E["type"]]: PrefixType<E>;
}[Event["type"]];
```

## 工具类型 Prettify

```typescript
export type Prettify<T> = {
  [K in keyof T]: Prettify<T[K]>;
} & {};
```
使用和对象求交集并且递归的方式，将交叉类型展开为平铺的对象类型，提升可读性。

## 类型编程

**泛型**：泛型被视为向类型系统中引入变量的手段。

**条件类型**：条件类型是 TypeScript 用来提供条件分支（conditional branch）的手段，这是人们之所以称它为图灵完备的原因之一。

### Infer 关键字
在通过条件类型判断输入类型是否满足给定条件（这本质上是检查它是否为目标类型的子类型）时，TypeScript 提供了 `infer` 关键字来让我们可以在这个过程中根据目标元素的泛型类型，提取出输入类型的额外信息。
```typescript
type Flatten<Type> = Type extends Array<infer Item> ? Item : Type;

type _1 = Flatten<string>;
//   ^? type _1 = string

type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : any;

type InstanceType<T extends abstract new (...args: any) => any> = T extends abstract new (...args: any) => infer R ? R : any;
```
- `new (...args: any) => any`: 这部分被称为“构造函数签名”（Construct Signature）。它描述了一个可以被 new 操作符调用的函数。
- `abstract` 关键字: 意味着泛型 T 不仅可以是一个能够被直接 new 的具体类（concrete class），也可以是一个不能被直接实例化的抽象类（abstract class）。

### 分配式条件类型
在通过 `extends` 关键字表达泛型的类型约束时，很多人都会忘记要检查的类型可能是联合类型这一事实。
当联合类型作为条件类型的输入时，TypeScript 会将联合类型中的每个类型单独拆出来计算条件类型，然后将得到的结果重新组合为联合类型。
```typescript
type ToArray<T> = T extends any ? T[] : never;
type _1 = ToArray<string | number>;
//   ^? type _1 = string[] | number[]
```

### 递归条件类型
在类型里面递归的调用自己：
```typescript
type Awaited<T> = T extends PromiseLike<infer U> ? Awaited<U> : T;
```
当 TypeScript 怀疑它碰到了无限递归（通过检查递归层数），或者一些需要花费很多时间计算的类型展开（比如对元组类型进行展开）时，它会停止类型检查并直接报错。
为了减少资源占用和防止 TypeScript 报错，我们最好将递归步骤处理成满足尾递归的形式。从 TypeScript 4.5 开始，它会对递归类型做尾递归优化。

### 使用 infer 关键字遍历元组
```typescript
type UppercaseArray<T extends string[]> =
  T extends [infer Head extends string, ...infer Tail extends string[]]
    ? [Uppercase<Head>, ...UppercaseArray<Tail>]
    : [];

type _1 = UppercaseArray<['foo', 'bar']>;
//   ^? type _1 = ["FOO", "BAR"]
```
字符串也可以遍历：
```typescript
type RemoveSpaces<T extends string> =
  string extends T // 👈 通过这种方式判断它是否为 string 类型
    ? string
    : T extends `${infer Head}${infer Tail}`
      ? `${Head extends ' ' ? '' : Head}${RemoveSpaces<Tail>}`
      : '';
```

### Type Parameter Inference
函数泛型可以让 TypeScript 从某个值中推导出类型，然后我们可以继续对这个类型进行计算：
```typescript
declare function setup<T>(config: { initial(): T }): T | null;

const _1 = setup({ initial() { return "seele" } });
//    ^? const _1: string | null
```
1. 根据传递的参数推导出源类型。
2. 解析方法签名的泛型 得到一个目标类型。
3. TypeScript 会发现源类型和目标类型都是对象类型，于是检查它们的属性。

### 约束检查与错误提示
```typescript
declare const ERROR_SYMBOL: unique symbol;
type MyTypeError<T extends string> = { [ERROR_SYMBOL]: T };

type DoCheck<T> = 'foo' extends keyof T
  ? unknown
  : MyTypeError<"对象没有包含必须的类型 foo 哦">;

function check<T>(input: T & DoCheck<T>) {
  return input;
}
check({}); // ERROR! 报错提示更友好
```

### 确保两个数组的长度相等
```typescript
// 当两个数组元素类型不同时，可以通过对返回值类型进行检查
declare function check2<T extends unknown[], U extends unknown[]>
  (a: [...T], b: [...U]): T['length'] extends U['length'] ? true : false;

// 对返回值类型进行检验不是解决问题的唯一的办法，我们还有：
type DoCheck<T extends unknown[], U extends unknown[]> =
  T['length'] extends U['length'] ? unknown : never;

declare function check3<T extends unknown[], U extends unknown[]>
  (a: [...T] & DoCheck<T, U>, b: [...U] & DoCheck<T, U>): void;

check3([1], ['foo']); // OK
```

### 函数类型闭包
函数类型闭包的作用是：把泛型参数传递到返回值类型里从而保留下来。
```typescript
type ValidatorMap = Record<string, NumberConstructor | StringConstructor | BooleanConstructor>;

type BuildResultType<T extends ValidatorMap> = {
  [K in keyof T]: ReturnType<T[K]>;
};

declare function buildValidatorFn<T extends ValidatorMap>
  (input: T): (input: Record<string, unknown>) => input is Prettify<BuildResultType<T>>;

const validatorFn = buildValidatorFn({ foo: String, bar: Number, baz: Boolean});
```

### 同态遍历类型 (Homomorphic Mapped Types)
这种类型指的是基于现有类型 产生新的类型。
1. **联合类型**：遍历联合类型的每一个类型 把每个成员作为新类型的键。
2. **数组/元组**：对其中的元素进行计算 返回计算结果构成的数组或者元组。
3. **字面量类型**：相当于只有一个元素的联合类型。
4. **原始类型**：返回原始类型本身。

```typescript
type Tuple = [key: string, value: unknown];

type getKeys<T extends Tuple[]> = {P in keyof T: T[P][0]}[number];

type arrToObject<T extends Tuple[]> = {
  [P in getKeys<T>]:
   T[number] extends infer U?
    U extends readonly [P, infer V] ? V : never
  : never
};
```
