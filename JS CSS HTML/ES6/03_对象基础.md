**对象的顺序？**

​	在我们遍历对象的时候，属性是按照什么顺序被遍历的呢？：先遍历数字属性 再是其他的

​	整数属性：按照升序排列

​	非整数属性：在ES6及以后，按照创建顺序排列

这个现象背后的原因是v8引擎基于性能的考量。
在js中 数组是一种特别的，索引是数字的对象。
我们通常对数组的操作都是按照一定的顺序 对数组进行遍历，那么js引擎就必须保证足够快的速度，所以js引擎会把这些索引存放在一块连续的内存空间 只需要对这块内存空间遍历即可。

但是 如果我们给相差极大的索引进行赋值 这种优化就不会生效 所以在操作数组的时，应该对连续的索引进行赋值 

对于非数值的常规属性，依赖于隐藏类来描述对象的结构 。
也就是我们按照特定的顺序给对象添加属性的时候 引擎会构建一个特性的形状的路径。
为了保持这种结构稳定可以复用（这样有相同属性添加顺序的对象就可以共享同一个隐藏类了）。属性就需要按照 插入顺序来记录。

### 对象本质

**对象的引用与复制**

和其他数据类型不一样，对象内部保存的是一个特定的内存地址，对象的本质是一个引用，直接复制对象的时候，复制的其实是那个引用。



**克隆与合并**

我们可以通过Object.assign（目标对象，[源对象数组]）进行浅层克隆，这个方法返回的是克隆好的目标对象。使用这个方法可以把很多对象的属性值合并到一个对象里面。如果要克隆的属性已经存在了，那么会覆盖。这个方法相当于遍历了一遍属性 复制给目标对象。

**深层克隆**

当对象的属性值也是一个对象的时候，我们需要深层克隆。

```javascript
/**
 * 功能完善的深克隆函数
 * @param {any} target - 需要克隆的目标
 * @param {WeakMap} map - 用于处理循环引用的缓存
 * @returns {any} - 克隆后的对象
 */
function deepClone(target, map = new WeakMap()) {
    // 1. 处理基本类型和 null
    if (typeof target !== 'object' || target === null) {
        return target;
    }

    // 2. 处理循环引用
    // 如果该对象已经被克隆过，直接返回缓存中的克隆对象
    if (map.has(target)) {
        return map.get(target);
    }

    // 3. 处理特殊对象类型
    const type = Object.prototype.toString.call(target);
    
    // Date
    if (type === '[object Date]') {
        return new Date(target);
    }
    
    // RegExp
    if (type === '[object RegExp]') {
        return new RegExp(target);
    }
    
    // Error
    if (type === '[object Error]') {
        return new Error(target.message);
    }

    // Set
    if (type === '[object Set]') {
        const cloneTarget = new Set();
        map.set(target, cloneTarget); // 先放入 map 防止循环引用
        target.forEach(value => {
            cloneTarget.add(deepClone(value, map));
        });
        return cloneTarget;
    }

    // Map
    if (type === '[object Map]') {
        const cloneTarget = new Map();
        map.set(target, cloneTarget); // 先放入 map 防止循环引用
        target.forEach((value, key) => {
            cloneTarget.set(deepClone(key, map), deepClone(value, map));
        });
        return cloneTarget;
    }

    // 4. 处理数组和普通对象
    
    // 保持原型链继承
    // 如果是数组，直接创建空数组
    // 如果是对象，使用 Object.create 保持原型链
    let cloneTarget;
    if (Array.isArray(target)) {
        cloneTarget = [];
    } else {
        cloneTarget = Object.create(Object.getPrototypeOf(target));
    }
    
    // 放入 map
    map.set(target, cloneTarget);

    // 处理所有属性 (包括 Symbol 和不可枚举属性)
    // Reflect.ownKeys = Object.getOwnPropertyNames + Object.getOwnPropertySymbols
    Reflect.ownKeys(target).forEach(key => {
        const desc = Object.getOwnPropertyDescriptor(target, key);
        // 通常只拷贝可枚举属性，或者全部拷贝
        if (desc && (desc.enumerable || key === 'length')) {
             cloneTarget[key] = deepClone(target[key], map);
        }
    });

    return cloneTarget;
}

// --- 测试用例 ---
/*
const original = {
    num: 1,
    date: new Date(),
    reg: /test/g,
    set: new Set([1, { a: 2 }]),
    map: new Map([['key', { b: 3 }]]),
    [Symbol('sym')]: 'symbolValue'
};
original.self = original; // 循环引用

const cloned = deepClone(original);
console.log(cloned.self === cloned); // true
console.log(cloned.date !== original.date); // true
*/
```
在这段代码中 使用了Reflect.ownKeys遍历数组，这个方法比使用fir object.keys更加强大
因为这个方法不仅可以获取索引属性 也能获取length属性 自定义属性 Symbol属性

总结：获取对象属性的方法：
- Object.keys(obj) ：最弱的。只拿 可枚举 的 字符串 键。
- Object.getOwnPropertyNames(obj) ：稍微强一点。能拿所有 字符串 键（包括不可枚举的），但还是 拿不到 Symbol 。
- Object.getOwnPropertySymbols(obj) ：专才。 只拿 Symbol 键。
- Reflect.ownKeys(obj) ： 字符串键、Symbol 键、不可枚举键，统统拿下 。

**垃圾回收**

可达值的基本集合（这些值被称为roots）：

- 正在执行的函数里面的局部变量
- 全局变量。
- 内置的全局变量
- 和DOM元素的属性相关的
- 绑定到DOM元素上面的事件监听器 
- 定时器的回调函数
- Promise对象内部的回调函数
- WeakMap WeakSet 这两种数据类型是弱引用，**它们不会阻止垃圾回收**。
  - 对于 WeakMap，键（Key）是弱引用的。如果一个对象只作为 WeakMap 的键存在，而没有其他强引用，那么垃圾回收器会回收这个对象，并且自动移除 WeakMap 中对应的键值对。
  - 对于 WeakSet，其存储的对象也是弱引用的。如果 WeakSet 中的对象没有其他引用，它们会被垃圾回收。
  - **注意**：WeakMap 的值（Value）是强引用的，只要键（Key）还存在，值就不会被回收。但是一旦键被回收，值也就无法访问了，最终也会被回收。
  
  **WeakMap 的实际用途**

  你可能会觉得这个特性没什么用，但实际上它解决了**内存泄漏**的大问题，特别是在处理 DOM 元素或第三方对象时。

  1.  **关联元数据（Metadata）**：
      假设你正在处理一个第三方的库或者 DOM 元素，你想给某个对象添加一些额外的数据（比如记录点击次数、缓存计算结果等）。
      - 如果用 `Map`：你需要手动管理这个 Map。当那个对象被销毁时，你需要记得去 Map 里删掉它，否则它永远不会被回收，导致内存泄漏。
      - 如果用 `WeakMap`：你不需要做任何事。当那个对象在其他地方被销毁（没有引用）时，WeakMap 会自动把它释放掉。

      ```javascript
      // 场景：记录每个用户的访问次数
      let visitCount = new WeakMap();

      function countUser(user) {
          let count = visitCount.get(user) || 0;
          visitCount.set(user, count + 1);
      }

      let john = { name: "John" };
      countUser(john); // John 访问了一次

      john = null; // John 离开了
      // WeakMap 会自动释放关于 John 的数据，不需要手动清理 visitCount
      ```

  2.  **缓存（Caching）**：
      当我们需要对一个函数的结果进行缓存，但又不希望缓存本身阻止对象被回收时。

  3.  **私有数据（Private Data）**：
      在 `class` 的私有字段（`#private`）出现之前，开发者常用 WeakMap 来存储实例的私有数据，这样外界无法直接访问，而且实例销毁后数据也会自动销毁。

  **为什么 Value 必须是强引用？**
  
  这其实是为了保证**数据的一致性**。
  
  试想一下，如果 Value 也是弱引用的：
  1.  你有一个对象 `user`（Key），和它的详细信息 `userInfo`（Value）。
  2.  你的程序正在用 `user`，准备通过 `weakMap.get(user)` 获取 `userInfo`。
  3.  突然，垃圾回收器（GC）运行了。
  4.  因为它觉得 `userInfo` 只有弱引用，就把 `userInfo` 回收了。
  5.  你的代码运行到 `weakMap.get(user)` 时，得到了 `undefined`。
  6.  **崩了。** 你的 `user` 还在，但它的数据却莫名其妙消失了。
  
  所以，`WeakMap` 的逻辑是：**只要 Key 还在，Value 就必须在。** Value 的命运是绑定在 Key 身上的。只有当 Key 死了（被回收），Value 才会失去保护，随之而去。




- 垃圾收集器找到所有的根，并“标记”（记住）它们。
- 然后它遍历并“标记”来自它们的所有引用。
- 然后它遍历标记的对象并标记 **它们的** 引用。所有被遍历到的对象都会被记住，以免将来再次遍历到同一个对象。
- ……如此操作，直到所有可达的（从根部）引用都被访问到。
- 没有被标记的对象都会被删除。

### 构造器

构造器函数在技术层面就是一个常规函数，只不过有一些约定俗成的规定：通过new操作符执行，以大写字母开头。

我们比较一下普通的执行一个函数，和使用new操作符执行一个函数有什么区别：

```javascript
//严格模式
function User(n){
	this.name = n
}
```

如果我们普通的执行这个函数，**函数里面没有this的概念**，这时还要往undefined里面添加属性，当然会报错不能往undefined里面添加属性了。

所以可以看出来，new操作符执行函数和普通执行函数的区别是，使用new操作符**会在函数里面隐式的创建一个this**，并且这个this是个空对象，然后我们就可以快乐的往空对象里面添加属性啦，而且最后它还会**隐式的返回这个this对象**。当然我们也可以理解成他把这个this的指向修改成了我们创建的那个对象，然后没有返回this的那一步，具体细节不重要！

如果我们想要这个构造器之使用一次，之后就无法被调用了，也可以这样写：

```javascript
/ 创建一个函数并立即使用 new 调用它
let user = new function() {
  this.name = "John";
  this.isAdmin = false;

  // ……用于用户创建的其他代码
  // 也许是复杂的逻辑和语句
  // 局部变量等
};
```

**构造器模式测试：new.target**

这个东西可以用在函数内部，判断函数是常规调用还是使用new操作符调用。

```javascript
function User(name) {
  if (!new.target) { // 如果你没有通过 new 运行我
    return new User(name); // ……我会给你添加 new
  }

  this.name = name;
}

let john = User("John"); // 将调用重定向到新用户
alert(john.name); // John
```


**构造器的return**

构造器是没有return的，但是如果非要return，那么：

1. 返回一个对象：那就返回这个对象，不管this
2. 返回原始类型或者空：这个return会被忽略



### **对象的原始值转换**

对象进行数学运算或者比较，会发生什么？

js并不允许自定义运算符对对象的处理方式，进行数学操作时，对象会被转换成原始值。

转换规则：

1. 没有转换成布尔值：所有的对象在布尔上下文里面都是真
2. 数字转换发生在对象相减或者应用数学函数的时候，比如Date对象详见可以得到两个日期的差值。
3. 字符串转换：比如alert一个对象，这个对象就变成字符串弹窗显示。



**hint**

那么js是怎样决定应用哪种规则的呢？

类型转换有三种变体，这个称为hint。

1. string:在执行alert，或者让对象作为另一个对象的键值时 
2. number：进行数学运算 或者通过< >进行比较的时候（这个比较运算符也可以是用于字符串，这里使用number是历史原因）
3. default：不确定期望值的类型的时候，比如加号既可以是数学运算符，也可以是字符串连接符 或者进行==比较的时候。

js通过一个内置方法[Symbol.toPrimitive]来进行类型转换，这个方法会得到一个参数hint，若hint为string，将会调用toString方法 会得到 [Object object] 如果得到的参数是number default 将会调用valueOf，如果这个方法不存在就还是会调用toString。

当然我们也可以重写这个方法：

```javascript
let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    alert(`hint: ${hint}`);
    return hint == "string" ? `{name: "${this.name}"}` : this.money;
  }
};
```

这些方法必须返回一个原始值，如果返回了一个对象 将会被忽略。

```javascript
let user = {name: "John"};

alert(user); // [object Object]
alert(user.valueOf() === user); // true
```

但是因为历史原因，这里的valueOf方法就是会返回对象本身，所以忽略他吧。

PS:这个valueOf方法一般是用于string对象，bollean对象，日期对象，可以获取字面量，所以一般普通的对象并不会内置，如果调用了这个方法，就会直接返回这个对象本身。

硬要实现自定义转换，其实可以重写这些方法：

```javascript
let user = {
  name: "John",
  money: 1000,

  // 对于 hint="string"
  toString() {
    return `{name: "${this.name}"}`;
  },

  // 对于 hint="number" 或 "default"
  valueOf() {
    return this.money;
  }
};

alert(user); // toString -> {name: "John"}
alert(+user); // valueOf -> 1000
alert(user + 500); // valueOf -> 1500
```

需要注意的是，转换的结果之要求是原始类型，至于是哪个原始类型就没有关系了。我们也可以只实现一种方法，统一处理。



**进一步的转换**

就比如在对对象执行乘法的时候，对象先是会进行原始值转换，然后生成的原始值将会进一步转换  转换成数字。

### 属性标志 属性描述符

```

Setter
```



**属性标志**

我们以前知道对象的属性就是键值对:key 和value 但是除了value 还有另外的三个属性:

1. writable:为真 则该属性可以修改 在严格模式下修改只读属性将会报错
2. enumerable:为真 则该属性会在循环里面被列出
3. configurable:为真 则该属性可以删除 否则不能删除 不能再次配置

不过如果是可写 不可配置 那么依然可以更改:

```javascript
let user = {
  name: "John"
};

Object.defineProperty(user, "name", {
  configurable: false
});

user.name = "Pete"; // 正常工作
delete user.name; // Error
```



当我们使用普通的方式 创建一个对象属性的时候,以上属性均为真.


通过`Object.defineProperty`修改属性描述符:如果这个属性存在,则修改,否则创建该属性.如果没有提供属性标识符,那么所有标志都是false

### 属性getter setter

对象属性有两种类型:数据属性 在之前用的所有属性都是数据属性 还有一种 **访问器属性** 本质其实是设置 访问属性值的函数.访问器的设计思想就是它看起来就是普通属性 但是不以函数调用的方式执行,而是我们正常读取属性 函数在幕后执行.

访问器由getter setter方法表示,在对象字面量里由get set表示



**访问器属性的描述符**

对于访问器属性是没有value和writeable的,所以当我们通过defineProperty定义一个属性的时候 可以:

```javascript
Object.defineProperty(user, 'fullName', {
  get() {
    return `${this.name} ${this.surname}`;
  },

  set(value) {
    [this.name, this.surname] = value.split(" ");
  }
});
```

只定义getter函数的时候，这个属性就成为一个只读属性。只定义setter属性的时候，这个属性就成为一个只设属性

这个只设属性看起来似乎没什么用，其实可以用来修改内部状态，而且不直接暴露这些状态：

```javascript
const obj = {
  _count: 0,
  set count(value) {
    this._count = value;
    console.log(`Count has been set to ${this._count}`);
  }
};

obj.count = 10; // 输出："Count has been set to 10"
```





**使用getter setter作为数据属性的包装器**

比如 我们要对某个数据属性进行限制 这个限制我们可以通过setter实现:

````java

      return;

  }
````

## JSON

在过去想把对象显式转换成字符串，可能需要自定义一些方法，为了简单，我们现在可以直接转换成JSON字符串。

JSON 是语言无关的纯数据规范



**JSON.stringfy**（val，[replacer,space]）:将对象 数组 字符串 数字 布尔 空转换成JSON字符串（这个字符串被称为序列化对象 JSON编码的对象 字符串化的对象 ）(也就是对象属性名使用双引号 字符串使用双引号 其他不变)。JSON字符串在js里面被认为是字符串

replacer是一个可选参数 可以是函数 或者属性列表 spaces代表空格数量
1. 如果是函数 那么 这个函数 **将会递归调用** **并且这个函数里面的this将会指向拥有这个值的对象** 这个函数将会接收两个参数 key value 
   1. 在这个函数里面 第一次key是空 value是需要序列化的对象 这里是做了一个处理把整个对象包装起来了 为了方便对这个对象整体进行一些处理
   
   ```javascript
   const user = {
       name: "John",
       age: 25,
       roles: {
           isAdmin: false,
           isEditor: true
       }
   };

   JSON.stringify(user, function(key, value) {
       // this 指向当前正在处理的属性所属的对象
       // 注意：第一次调用时，key 是空字符串，value 是整个 user 对象
       
       if (key === 'age') return value + 1; // 修改值：把年龄加 1
       if (key === 'isAdmin') return undefined; // 过滤值：不序列化该属性
       
       return value;
   });
   // 结果: {"name":"John","age":26,"roles":{"isEditor":true}}
   ```

2. 如果是一个数组 那么这个数组里面的元素就代表需要序列化的属性 对每个对象 **递归检查是不是拥有这些属性 如果有才会被序列化**

   ```javascript
   const user = {
       name: "Alice",
       friend: {
           name: "Bob", 
           age: 30
       }
   };
   
   // 数组作为白名单，不仅过滤第一层，也会递归过滤深层对象
   JSON.stringify(user, ['name', 'friend']);
   // 结果：{"name":"Alice", "friend":{"name":"Bob"}}
   // friend.age 被丢弃了，因为 'age' 不在白名单里
   ```

1. 由于JSON是一种数据规范 和语言无关 所以有一些js里面才有的属性在序列化的时候会被忽略 比如：函数属性 比如字符串的一些函数属性什么的 symbol类型的键和值 值为undefined的属性

## 对象进阶 (Advanced Object Concepts)

宝贝，既然你想要更深入地了解对象，Daddy 就带你看看对象内心深处最隐秘的角落。这些知识点能让你对对象的掌控力达到一个新的高度。

### 1. 真正的“冻结”：递归深冻结 (Deep Freeze)

你可能已经知道 `Object.freeze()` 可以让对象变得不可修改。但它只是**浅层**的。如果对象里还嵌套着对象，里面的那些还是可以被肆意修改的。

要想完全占有一个对象，让它永远保持初见时的模样，你需要**深冻结**：

```javascript
function deepFreeze(obj) {
  // 获取所有属性名（包括不可枚举的）
  const propNames = Reflect.ownKeys(obj);

  // 在冻结自身之前，先冻结所有属性
  for (const name of propNames) {
    const value = obj[name];

    // 如果属性值是对象或函数，且未被冻结，则递归冻结
    if ((value && typeof value === "object") || typeof value === "function") {
      if (!Object.isFrozen(value)) {
        deepFreeze(value);
      }
    }
  }

  return Object.freeze(obj);
}

const user = {
  name: "John",
  preferences: {
    theme: "dark"
  }
};

deepFreeze(user);
// user.preferences.theme = "light"; // 报错！现在它是彻底的冰山美人了。
```

### 2. 纯净之美：无原型对象 (Prototype-less Objects)

普通对象 `{}` 都会继承 `Object.prototype`，这意味着它们自带了 `toString`, `hasOwnProperty` 等方法。但在某些场景下（比如作为单纯的字典 Map 使用），这些继承属性可能会造成**原型污染**或键名冲突（比如用户正好有个键名叫 `toString`）。

我们可以创建一个**完全纯净**的对象：

```javascript
// 创建一个没有原型的对象
const pureObj = Object.create(null);

pureObj.name = "Daddy";
// alert(pureObj); // 报错！因为它连 toString 方法都没有
console.log(Object.getPrototypeOf(pureObj)); // null
```

这种对象非常适合用来存储不可信的用户输入数据，因为它没有任何预置的“后门”。

### 3. 定制对象的“身份证”：Symbol.toStringTag

你一定用过 `Object.prototype.toString.call(obj)` 来判断类型，通常它返回 `[object Object]`。
想让你的对象拥有专属的类型标签吗？用 `Symbol.toStringTag`。

```javascript
const user = {
  [Symbol.toStringTag]: "User"
};

console.log(Object.prototype.toString.call(user)); // [object User]
```

这不仅很酷，在调试和类型检查时也非常有用。

### 4. 让对象动起来：Symbol.iterator

普通对象是不能用 `for...of` 遍历的，只能用 `for...in`。
但只要你给它装上 `Symbol.iterator`，它就能像数组一样被迭代。

```javascript
const range = {
  from: 1,
  to: 5,

  [Symbol.iterator]() {
    let current = this.from;
    let last = this.to;

    return {
      next() {
        if (current <= last) {
          return { done: false, value: current++ };
        } else {
          return { done: true };
        }
      }
    };
  }
};

for (let num of range) {
  console.log(num); // 1, 2, 3, 4, 5
}
// 甚至可以使用扩展运算符
console.log([...range]); // [1, 2, 3, 4, 5]
```

### 5. defineProperty 的“默认陷阱”

当你直接用 `obj.key = value` 赋值时，属性的标志（writable, enumerable, configurable）默认都是 `true`。
但当你用 `Object.defineProperty` 创建新属性时，它们的默认值全是 **`false`**。

```javascript
const obj = {};
Object.defineProperty(obj, "hidden", {
  value: "secret"
});

// obj.hidden = "changed"; // 失败！不可写
// console.log(Object.keys(obj)); // [] 空的！不可枚举
// delete obj.hidden; // 失败！不可配置
```

所以，使用 `defineProperty` 时，如果要让属性像正常属性一样，记得要把那些标志手动设为 `true`。

### 6. 真理之眼：Object.is



```javascript
Object.is(NaN, NaN); // true
Object.is(+0, -0);   // false
```

虽然日常用到的机会不多，但在处理一些极度精确的数学计算或者 React 的 shallow compare 时，它可是幕后的大功臣。

### 7. 更安全的检查：Object.hasOwn

以前我们要检查对象自身是否有某个属性，总是用 `obj.hasOwnProperty('key')`。
但是，如果这个对象是 `Object.create(null)` 创建的，它就没有这个方法，代码就会报错崩掉。
或者，如果有个坏蛋给对象加了个属性叫 `hasOwnProperty`，覆盖了原有的方法，那就更乱了。

建议你用 `Object.hasOwn(obj, 'key')`，它相当于 `Object.prototype.hasOwnProperty.call(obj, 'key')` 的缩写，既安全又方便。

```javascript
const obj = Object.create(null);
obj.name = "Daddy";
// obj.hasOwnProperty("name"); // 报错！
console.log(Object.hasOwn(obj, "name")); // true
```

### 8. 对象的逆向工程：Object.fromEntries

你肯定知道 `Object.entries(obj)` 可以把对象变成 `[key, value]` 的数组。
那如果我们想变回来呢？比如你把对象转换成数组，用 `map` 处理了一番，想再变回对象。
这时候就用 `Object.fromEntries`。

```javascript
const prices = {
  banana: 1,
  orange: 2,
  meat: 4,
};

// 想要把所有价格翻倍
const doublePrices = Object.fromEntries(
  Object.entries(prices).map(([key, value]) => [key, value * 2])
);

console.log(doublePrices); // { banana: 2, orange: 4, meat: 8 }
```

它也非常适合把 `Map` 转换成普通对象。

### 9. 对象里的 super？

是的，`super` 关键字不仅仅属于类（Class）。在对象字面量里，如果你使用简写的方定义法，也可以使用 `super` 来访问原型链上的方法。

```javascript
const animal = {
  eat() {
    return "Nom nom";
  }
};

const rabbit = {
  __proto__: animal,
  eat() {
    // 调用父类方法
    return super.eat() + " ...Chomp!";
  }
};

console.log(rabbit.eat()); // Nom nom ...Chomp!
```

**注意**：这只在使用 `method() {}` 简写语法时有效。如果你写成 `method: function() {}`，`super` 就无法工作了，因为 `super` 需要绑定到方法的 `[[HomeObject]]` 属性上，只有简写语法会生成这个属性。

### 深入理解：[[HomeObject]]


你可以把它理解为函数的**“出生证明”**或者**“户口本”**。

#### 1. 它是什么？
当你在对象或类中使用**简写语法**（`method() {}`）定义一个函数时，JavaScript 引擎会自动在这个函数对象上添加一个内部属性 `[[HomeObject]]`。
**这个属性的值，就是该方法被定义的那个对象。**

#### 2. 为什么要用它？
是为了解决 `super` 的指向问题。
你可能会想，`super` 不就是找当前对象的原型吗？用 `this.__proto__` 不行吗？
**不行！** 因为 `this` 是动态的。

看个例子：

```javascript
let animal = {
  eat() {
    return "Animal eat";
  }
};

let rabbit = {
  __proto__: animal,
  eat() {
    return super.eat(); // 这里 super 需要知道“我是谁的方法”，才能去我的原型找
  }
};

let longEar = {
  __proto__: rabbit,
  eat() {
    return super.eat();
  }
};

// 执行 longEar.eat()
// 1. longEar.eat() 里的 super 找到了 rabbit.eat()
// 2. rabbit.eat() 被调用，此时 this 依然是 longEar
// 3. 如果 rabbit.eat() 里用 this.__proto__ 来找父类方法...
//    this.__proto__ 是 rabbit
//    结果就是：它又调用了 rabbit.eat()！
//    死循环了！😱
```

**`[[HomeObject]]` 的救场：**
因为 `rabbit.eat` 这个函数身上带着 `[[HomeObject]]`（指向 `rabbit` 对象）。
不管 `this` 指向谁，当执行到 `super` 时，引擎会说：
“好的，这个函数的家是 `rabbit`，那我要去 `rabbit` 的原型（`animal`）那里找 `eat` 方法。”
这样就完美避开了 `this` 带来的死循环陷阱。

#### 3. 只有简写语法才有
这就是为什么 `method: function() {}` 这种写法不能用 `super`。因为这种普通函数定义方式，引擎**不会**给它设置 `[[HomeObject]]`。它就像个流浪汉，不知道自己的家在哪，自然也就找不到“家长的家长”（原型）了。

### 10. 性能陷阱：delete 操作符


V8 会为对象创建“隐藏类”（Hidden Classes）来优化属性访问速度。如果你动态地 `delete` 一个属性，会破坏这个隐藏类的转换链，导致对象退化成哈希表存储模式，访问速度变慢。

**建议**：
如果你非常在意性能（比如在循环里处理百万级对象），尽量不要用 `delete`。
你可以把属性设置为 `null` 或 `undefined` 代替删除：`user.name = null`。
或者，如果你真的要删除，不如重组一个新对象（使用解构或 `Object.fromEntries` 过滤掉不需要的属性）。

### 11. 安全陷阱：原型污染 (Prototype Pollution)

这是一个很坏的漏洞。如果你的代码允许攻击者控制对象的键名（比如处理 JSON 或者合并对象时），他们可能会把键名设为 `__proto__`。

```javascript
const userInput = '{"__proto__": {"isAdmin": true}}';
const obj = {};
const malicious = JSON.parse(userInput);

// 危险的合并函数
function merge(target, source) {
  for (let key in source) {
    target[key] = source[key];
  }
}

merge(obj, malicious);
// 如果没有防范，这可能会修改 Object.prototype
// 导致所有空对象都莫名其妙有了 isAdmin = true
```

**如何防范？**
1. 使用 `Object.create(null)` 创建没有原型的对象。
2. 在合并对象时，过滤掉 `__proto__`, `constructor`, `prototype` 这些敏感键名。
3. 使用 `Object.freeze(Object.prototype)`（虽然这可能会破坏某些库）。
