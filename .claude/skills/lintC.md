# lintC - Git 提交助手

执行 git add 并分析文件变更，自动生成规范的提交信息并推送。

## 执行步骤

1. **添加所有变更** - 运行 `git add .`
2. **分析变更内容** - 使用 `git diff --cached` 和 `git status` 分析本次变更
3. **生成提交信息** - 根据变更类型自动生成：
   - `add: 增加了...` - 新增文件或功能
   - `del: 删除了...` - 删除文件或移除功能
   - `refactor: 对...进行更改` - 重构、优化、调整
   - `feat: ...` - 仓库建设相关（AI上下文、skill配置、Actions配置等）
   - `fix: ...` - 修复问题
4. **提交变更** - 执行 `git commit`
5. **推送到远程** - 执行 `git push`

## 提交类型判断规则

| 变更情况 | 提交类型 | 示例 |
|---------|---------|------|
| 新增文件 | add | `add: 增加了 ES6异步编程笔记` |
| 删除文件 | del | `del: 删除了过时的配置文件` |
| 修改现有文件内容 | refactor | `refactor: 更新了 Vue响应式原理章节` |
| 修改 .claude/、.trae/、.github/ | feat | `feat: 配置了 AI 目录生成 Action` |
| 修复 bug 或问题 | fix | `fix: 修复了字数统计脚本路径问题` |

## 多类型变更处理

如果变更涉及多种类型，按优先级选择主要类型：
1. feat（仓库建设）
2. fix（修复）
3. add/del/refactor（内容变更）

或者按变更占比生成复合提交信息，如：
```
refactor: 更新了 Vue 笔记，增加了 React Fiber 章节
```

## 注意事项

- 不要提交敏感信息（.env、credentials等）
- 确保 .gitignore 配置正确
- 如果 push 失败，先处理远程冲突再重试
- 提交信息保持简洁，不超过 50 字
